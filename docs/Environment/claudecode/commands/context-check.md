以下の手順でコンテキストウィンドウを分析・最適化してください。

## Step 1: 現状確認
/context コマンドを実行して現在のコンテキスト使用状況を取得してください。
/doctor コマンドも実行して、MCPサーバーごとのトークン消費を確認してください。

## Step 2: 内訳の整理
取得した情報を以下の表形式でまとめてください。

| 項目 | トークン数 | 使用率 | 判定 |
|------|-----------|--------|------|
| System prompt | - | - | 固定 |
| System tools | - | - | 固定 |
| MCP tools | - | - | 要確認 |
| Memory files | - | - | 要確認 |
| Messages | - | - | 要確認 |
| Reserved | - | - | 固定 |
| Free space | - | - | - |

判定基準：
- 「固定」= Claude Code本体の必須コスト、削減不可
- 「要確認」= 削減できる可能性あり
- Free spaceが30%を下回ったら警告

## Step 3: 問題項目の特定と解決案
使用率が高い「要確認」項目それぞれについて、以下の観点で解決案を提示してください。

### MCP toolsが多い場合
- /mcp で接続中のサーバー一覧を確認
- 今のセッションで使っていないサーバーを特定
- 無効化の具体的なコマンドを提示

### Memory files (CLAUDE.md) が多い場合
- どのファイルが何トークン消費しているか確認
- 今のタスクに不要な記述がないか提案
- スキルファイルへの移行が適切な箇所を指摘

### Messages (会話履歴) が多い場合
- /compact の実行タイミングを判断
- 圧縮前に保持すべき重要決定事項があれば確認

## Step 4: アクションサマリー
「今すぐできること」「後でできること」に分けて優先順位をつけて提示してください。

今すぐできること（このセッション中に対処可能）：
1. ...

後でできること（設定変更で恒久対応）：
1. ...

## 注意事項
- 削減提案は保守的に行うこと（よく使うツールは外さない）
- 各提案に期待削減トークン数の目安を添えること
- 不明な点は推測せず、確認を促すこと
