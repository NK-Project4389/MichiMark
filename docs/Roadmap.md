# MichiMark ロードマップ

最終更新: 2026-04-11

---

## プロダクトビジョン

> 時間 × 場所 × アクションのストーリーを記録し、欲しい情報に集計・可視化して、使う人の生産性を向上させる。

---

## ターゲットセグメント

| グループ | セグメント | コアバリュー |
|---|---|---|
| **趣味ログ系** | ツーリング・ロードバイク・ランニング・ドライブ・散歩 | 「あの日どこ行って何したか」を残したい |
| **旅・おでかけ系** | 旅行計画者 | 旅程記録・旅費管理 |
| **仕事移動系** | 訪問介護・暮らしのマーケット出店者 | 移動記録・稼働管理・精算 |

全セグメントに共通：**移動 × 時間 × 記録への欲求**

## マネタイズ戦略

### モデル：Freemium + サブスクリプション

| プラン | 対象 | 単価（月） | 主な機能 |
|---|---|---|---|
| **無料** | 個人 | ¥0 | 記録無制限・直近7日サマリー・イベントOverview |
| **個人プラン** | 個人・フリーランス | ¥300 | 月次・任意期間サマリー・CSVエクスポート |
| **チームプラン** | 中小企業 | ¥500 / 台 | 個人プラン全機能 + 管理者ダッシュボード・車両管理（2〜5台） |
| **法人プラン** | 自社車両6台〜 | ¥400 / 台 | チームプラン全機能 + ボリューム割引（6〜20台） |

※ 20台超は運行管理システムの領域としてスコープ外
※ 年払い：2ヶ月分割引

### 転換ポイント設計

```
無料ユーザーが1ヶ月使う
  → データが溜まる
  → 月次サマリーを見ようとする → 個人プラン案内（¥300転換ポイント）
  → 従業員・車両が増える → チームプラン案内（アップセルポイント）
```

### スコープ定義

- **対象：自社車両の管理**（自社ドライバー × 自社車両）
- **対象外：受託・外注関係**
- **対象外：運行管理法対応**（点呼・緑ナンバー事業者向け機能は含まない）

### Phase D：マーケティング本格展開

**目標：ユーザー獲得を加速させる。マーケターAgentが主役。**

#### マーケターAgent 役割定義

```
marketer
  ├─ サクセスストーリー作成（ターゲット別・全8本）
  ├─ SNS投稿文（X・Instagram・Facebook）
  ├─ App Store テキスト（タイトル・説明文・キーワード）
  └─ デザイナーへの要件書（スクショ・バナー・アイコン方針）
```

**アウトプット形式：**

| 出力物 | 形式 | 用途 |
|---|---|---|
| SNS投稿文 | `.txt` | そのままコピペで投稿可能 |
| App Store テキスト | `.txt`（文字数制限準拠） | そのままApp Store Connectに貼り付け |
| デザイン要件書 | `.md` | designerへの叩きとして渡す |

#### サクセスストーリー（ターゲット別・8本）

| # | ターゲット | Before → After の軸 |
|---|---|---|
| 1 | ツーリング | 「どこ行ったか忘れる」→「ルートと時間が全部残る」 |
| 2 | ロードバイク | 「走行データがバラバラ」→「距離・時間をひとまとめ」 |
| 3 | ランニング | 「コースのメモが面倒」→「走りながら記録できる」 |
| 4 | ドライブ趣味 | 「ドライブの記憶が薄れる」→「写真＋記録でストーリーに」 |
| 5 | 散歩 | 「歩いた道を覚えていない」→「散歩ログが地図になる」 |
| 6 | 旅行計画者 | 「旅費の計算が後でバタバタ」→「移動中にリアルタイム記録」 |
| 7 | 暮らしのマーケット出店者 | 「交通費・出店費の管理が曖昧」→「イベントごとに収支が見える」 |
| 8 | 訪問介護 | 「訪問記録を手書き」→「移動ログと稼働時間が自動集計」 |
