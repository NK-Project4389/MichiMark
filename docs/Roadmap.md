## Phase 2 — トピック機能（定型タイプ提供）

**目標：用途別にイベントの入力項目・タブ・アクション種別を最適化する**

### 概念モデル

```
Topic
  ├─ id / name / type（定型 or カスタム）
  ├─ 表示タブ設定（タブごとに visible: true/false）
  └─ 利用可能アクション種別
```

イベントにトピックを紐づけることで、EventDetail の表示タブが切り替わる。

### 定型トピック（Phase 2 提供予定）

| トピック | コアバリュー | 主な表示タブ |
|---|---|---|
| **作業訪問** | 移動 / 作業 / 休憩の時間記録 | basic_info, michi_info, fuel_detail |
| **営業・車移動** | 走行距離 + ガソリン代精算 | basic_info, michi_info, fuel_detail, payment_detail, payment_info |
| **旅行** | 旅費・経費の管理 | basic_info, michi_info, payment_detail, payment_info |

※ タブ構成の詳細は各トピックの要件書で定義する

### タスク

| タスク | 概要 |
|---|---|
| Topic ドメインモデル設計 | Topic エンティティ・タブ設定の設計 |
| トピック選択 UI | イベント作成時にトピックを選択する画面 |
| EventDetail タブ動的制御 | トピックに応じたタブ表示 / 非表示の切り替え |
| 定型トピック 3種 実装 | 作業訪問 / 営業・車移動 / 旅行 |

## Phase 3 — アクション × 時間モデル

**目標：アクションに状態遷移と時間計測を持たせ、フィールドワーカーの活動を記録できるようにする**

### 概念モデル

```
Action
  ├─ 単体型：発生したことを記録（フェーズ変化なし）
  └─ 状態遷移型：現在のフェーズを切り替えるポイント
       例）出発 → [移動中] → 到着 → [作業中] → 完了

Phase（状態）
  ├─ リアルタイムタイマー（開始 / 終了）
  └─ サブアクション（トグル型）
       例）作業中フェーズ内の「休憩」トグル
           ON = 休憩開始 / OFF = 休憩終了
           ※ フェーズ自体は「作業中」のまま継続
```

### タスク

| タスク | 概要 |
|---|---|
| Action ドメインモデル拡張 | 状態遷移型 / 単体型の設計 |
| Phase（状態）モデル設計 | フェーズ管理・タイマーの設計 |
| サブアクション（トグル）設計 | フェーズ内ネストアクションの設計 |
| リアルタイムタイマー UI | 計測中の表示・開始 / 停止操作 |
| 場所の手動登録 | アクション発生時の場所を手動入力（GPS は Phase 5 以降） |
