# 1. Feature Overview

## Feature Name

DashboardTabRename（ダッシュボードタブ名変更）

## Purpose

ダッシュボード画面のトップレベルナビゲーションバーにある「イベント」タブラベルを「イベント一覧」に変更し、タブの役割をユーザーが直感的に理解できるようにする。あわせて EventListPage の AppBar タイトルも統一する。

## Scope

含むもの
- ボトムナビゲーションのタブラベル「イベント」→「イベント一覧」変更
- EventListPage の AppBar タイトル「イベント」→「イベント一覧」変更（現状「イベント」と表示されている場合のみ）

含まないもの
- タブのアイコン・順序・その他のタブ名
- ナビゲーションルートのパス名・識別子（変更しない）
- データモデル・Domain・Repository・Bloc のロジック（変更しない）
- ダッシュボード以外の画面に存在するイベント関連テキスト
- 多言語対応（ローカライズ）の整備

---

# 2. Feature Responsibility

変更対象は表示文字列のみ。

- Draft / Projection / Bloc / Domain / Router の変更なし
- `_ScaffoldWithBottomNav` のタブラベル文字列のみ変更する
- `EventListPage` の AppBar タイトル文字列のみ変更する（Widgetレベルの変更）

---

# 3. 変更内容

## 3-1. 表示文字列変更一覧

| 変更箇所 | 変更前 | 変更後 |
|---|---|---|
| ボトムナビゲーション イベントタブラベル | イベント | イベント一覧 |
| EventListPage AppBar タイトル | イベント（該当する場合） | イベント一覧 |

## 3-2. Widget Key

変更なし。既存のキーをそのまま維持する。

| Widget | Key |
|---|---|
| イベント一覧タブ | `Key('event_list_tab')` |

> キーが未付与の場合は UI-17（FS-dashboard_tab_position）での対応時に付与される。

---

# 4. Data Flow

変更なし（表示文字列のみ変更のため）

---

# 5. Router変更方針

変更なし（パス名・識別子に変更なし）

# 6. Test Scenarios

## 前提条件

- iOSシミュレーター起動済み
- アプリが起動できること

## テストシナリオ一覧

| ID | シナリオ名 | 種別 | 優先度 |
|---|---|---|---|
| TC-RNM-001 | ナビゲーションバーに「イベント一覧」タブラベルが表示される | Integration | High |

## シナリオ詳細

### TC-RNM-001: ナビゲーションバーに「イベント一覧」タブラベルが表示される

**前提:**
- アプリを起動した状態

**操作手順:**
1. アプリを起動する

**期待結果:**
- ナビゲーションバーに「イベント一覧」というテキストが表示されること
- `find.text('イベント一覧')` で1件以上ヒットすること

**実装ノート（ウィジェットキー一覧）:**
- `Key('event_list_tab')`: イベント一覧タブ（ナビゲーションバー）
- `find.text('イベント一覧')` でラベルテキストを検証する

---

### TC-RNM-002: ナビゲーションバーに「イベント」（単独表記）が残っていない

**前提:**
- アプリを起動した状態

**操作手順:**
1. アプリを起動する
2. ナビゲーションバーのラベルを確認する

**期待結果:**
- ナビゲーションバー上に `find.text('イベント')` でヒットするウィジェットが存在しないこと
  - ただし「イベント一覧」の部分文字列マッチを除くため、厳密に `'イベント'`（単独）を検証すること

**実装ノート（ウィジェットキー一覧）:**
- `Key('event_list_tab')`: イベント一覧タブ（ナビゲーションバー）
- `find.text('イベント')` は0件であること（`findsNothing`）
- ただしナビゲーションバーの中のみ検索対象とする（EventDetailなど他画面のテキストは対象外）

---

### TC-RNM-003: イベント一覧画面の AppBar タイトルが「イベント一覧」である

**前提:**
- アプリ起動後にイベント一覧タブを表示している状態

**操作手順:**
1. アプリを起動する
2. `Key('event_list_tab')` をタップしてイベント一覧画面を表示する
3. AppBar タイトルを確認する

**期待結果:**
- AppBar に「イベント一覧」というテキストが表示されること

**実装ノート（ウィジェットキー一覧）:**
- `Key('event_list_tab')`: イベント一覧タブ（ナビゲーションバー）
- `find.text('イベント一覧')` が AppBar 内に表示されること

---

### TC-RNM-004: タブの選択・切り替え動作に影響がない

**前提:**
- アプリを起動した状態

**操作手順:**
1. アプリを起動する
2. `Key('event_list_tab')` をタップする
3. `Key('dashboard_tab')` をタップする

**期待結果:**
- イベント一覧タブ・ダッシュボードタブの切り替えが正常に動作すること
- タブ切り替え後に対応する画面が表示されること

**実装ノート（ウィジェットキー一覧）:**
- `Key('event_list_tab')`: イベント一覧タブ（ナビゲーションバー）
- `Key('dashboard_tab')`: ダッシュボードタブ（ナビゲーションバー）
