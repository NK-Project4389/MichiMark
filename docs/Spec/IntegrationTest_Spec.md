# MichiMark Integration Test 仕様書

Version: 1.0
作成日: 2026-04-10
目的: 本番リリース前の最終確認を目的としたIntegration Testのシナリオ設計書。機能軸でグループ化し、計算ロジック・データ整合性・画面遷移を優先的に検証する。

---

## 方針

- **削除対象**: 色・フォントスタイル・ウィジェット存在の確認のみのテスト（Widget Test / Unit Testの範疇）
- **残す対象**: ユーザー操作を起点とした機能確認・計算結果確認・データ整合性確認
- **統合対象**: 同一操作フローで複数の選択肢を検証するケースは1シナリオにまとめる

### TC-EVT-001: EventDetail保存後EventListに変更が反映される

**前提:** event-001「箱根日帰りドライブ」が一覧に表示されている

**操作手順:**
1. EventListで「箱根日帰りドライブ」をタップ
2. 概要タブの BasicInfo セクション（`Key('basicInfoRead_container_section')`）をタップして編集モードへ
3. イベント名を「テスト変更イベント名_EVT001」に変更
4. 「保存」をタップ
5. 戻るボタンをタップしてEventListに戻る

**期待結果:**
- EventListに「テスト変更イベント名_EVT001」が表示される

---

### TC-EVT-002: 新規イベント作成後EventListにイベントが追加される

**前提:** EventListが表示されている

1. EventListのFABをタップ

3. EventDetailが開いたことを確認
4. 戻るボタンをタップしてEventListに戻る

**期待結果:**
