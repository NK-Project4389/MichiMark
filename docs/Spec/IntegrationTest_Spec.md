# MichiMark Integration Test 仕様書

Version: 1.0
作成日: 2026-04-10
目的: 本番リリース前の最終確認を目的としたIntegration Testのシナリオ設計書。機能軸でグループ化し、計算ロジック・データ整合性・画面遷移を優先的に検証する。

---

## 方針

- **削除対象**: 色・フォントスタイル・ウィジェット存在の確認のみのテスト（Widget Test / Unit Testの範疇）
- **残す対象**: ユーザー操作を起点とした機能確認・計算結果確認・データ整合性確認
- **統合対象**: 同一操作フローで複数の選択肢を検証するケースは1シナリオにまとめる

### TC-PAY-003: 精算金額が均等割り（端数切り捨て）で正しく計算される

**前提:** event-002「富士五湖キャンプ」を開いて概要タブへ移動。

シードデータの計算根拠:
- 高速道路代 ¥4,500 ÷ 3名 = **¥1,500**（割り切れる）
- キャンプ場利用料 ¥8,000 ÷ 3名 = **¥2,666**（端数切り捨て）
