### 確定した主要決定事項

#### マネタイズモデル

| プラン | 単価 | 主な有料機能 |
|---|---|---|
| 無料 | ¥0 | 記録無制限・直近7日サマリー・イベントOverview |
| 個人プラン | ¥600/月 | 月次・任意期間サマリー・CSVエクスポート |
| チームプラン | ¥400/台/月 | 管理者ダッシュボード・車両管理 |
| 法人プラン | ¥350/台/月 | ボリューム割引 |

- 転換ポイント：1ヶ月使用後の月次サマリーアクセスでサブスク案内
- CSVエクスポートはサブスクに同梱（単体有料化はしない）
- 記録は無制限（件数制限なし）・直近7日サマリーは無料

#### スコープ定義

- **対象：自社車両の管理**（自社ドライバー × 自社車両）
- **対象外：受託・外注関係**（A社がB社の車両を管理するケースは含まない）
- 理由：Organization を1社完結に保ちシステム複雑性を抑制する

#### 新機能ロードマップ

- **Phase 2**：トピック機能（定型3種：作業訪問 / 営業・車移動 / 旅行）
  - トピックはイベントの属性
  - トピックによってEventDetailの表示タブが切り替わる
- **Phase 3**：アクション × 時間モデル（状態遷移・リアルタイムタイマー・サブアクション）
- **Phase 4**：集計・可視化・CSVエクスポート・チームプラン
- **Phase 5**：カスタムトピック・GPS・クラウド同期・Web管理画面

#### drift実装前に対処すべき設計考慮事項

| 項目 | 内容 |
|---|---|
| UUIDへの変更 | 全エンティティIDをauto-increment → UUID（String）に変更 |
| Organization / Vehicle エンティティ追加 | ドメインモデルに組織・車両を追加 |
| Event に user_id / vehicle_id 追加 | nullable フィールドとして追加 |
| Repository インターフェース抽象化 | クラウド同期を前提としたI/F設計 |
| AuthContext の DI 確保 | userId / organizationId / role を持たせる場所を確保 |
