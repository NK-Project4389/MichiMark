色彩設計の考え方: Mark (地点) には既存ブランドカラーの Teal (#2B7A9B) をそのまま使用し、 Link (区間) には既存パレットの Emerald Green (#2E9E6B) を採用しています。 これはトポグラフィックに隣り合う色（H=155° と H=200°）を使い、 青緑系の統一感を維持しながら Mark/Link を明確に区別する設計です。 色相差は 45° で、近すぎず遠すぎないバランスを持ちます。

設計根拠

各デザイン判断の理由と MichiMark のデザイン原則との対応をまとめました。

✅

推奨: パターン C（タイムライン連結型）

MichiMark のコアコンセプト「道を記録する」を最も直感的に表現するのはタイムライン連結型です。 縦線・ドット・カードが一体となって「ルートの流れ」を構成し、 スクロールするだけでドライブの時系列を追える体験を実現します。 また既存の REQ-michi_info_layout.md が定義した「縦線 + ドット + 吹き出しカード」設計とも方向性が一致しており、 実装上の整合性も高いです。REQ-michi_info_timeline_redesign.md の「罫線接続」への変更案とも自然に組み合わせられます。

🌻 Mark = Teal, Link = Emerald の理由

Teal (H=200°) は既存ブランドカラーそのもの。Mark は「地点 = 記録の核」であり アプリのアイデンティティカラーを使うことで重要性を表現します。 Emerald (H=155°) は Teal から色相 -45° の隣接色で、 統一感を保ちながら明確に区別できる組み合わせです（既存の movingCost カラーとの共有も可）。 色相環上で近い青緑系を使うことで「同じドライブ記録の一部」というグルーピング感も生まれます。

● Mark = 円ドット, Link = 角丸矩形ドットの理由

円形は「地点・ピン」を普遍的に表すアイコン言語（地図のマーカー）と一致します。 Mark が地点を表すため円形ドットが直感的です。 Link は「区間・線分」という面的な概念を表すため、 角丸矩形（ダッシュのような形）を使うことで「連結・移動」のニュアンスを持たせています。 この差異は色彩差に加えて形状差も加わり、視覚的多様性でより確実な区別を実現します。

📷 縦ライン配色（Teal/Emerald）の根拠

タイムライン縦線のカラーを Mark/Link の区間に応じて Teal/Emerald で切り替えることで、 縦線自体が「今どこを見ているか」のコンテキスト情報を提供します。 Link 区間で縦線が Emerald になることで、その間が「移動中」であることが カードを読まなくても縦線の色変化で把握できます。これは高速スクロール時のUXを向上させます。

📃 Link カード = Tint 背景の理由

パターン C では Link カードの背景を Emerald Tint (#EDFAF4) にしています。 Mark カードが白背景・上ボーダー強調なのに対し、 Link カードが薄いグリーン背景を持つことで、 リスト上での視覚的ウェイト（重み）を Mark > Link とし、 地点が主役・区間が補助という情報階層を色で表現します。 Tint は彩度が低く（S≈30%）、長いリストでも目が疲れにくいです。

📹 モバイルサイズ設計の根拠

Mark カード高さ 64dp・Link カード高さ 52dp は、 iOS の最小タッチターゲット 44pt・Android の 48dp 以上を確保しながら 1画面に 5～7 アイテムを表示できるサイズです。 REQ-michi_info_timeline_redesign.md が定義する _cardHeight = 72.0 との整合を取りつつ、 Link カードをやや小さくすることで Mark の重要性を階層的に示します。

🌎 WCAG AA 対応の考え方

全カラーペアのコントラスト比を 4.5:1 以上（WCAG AA）で設計しています。 特に距離値などの重要な数値情報は Teal Primary / Emerald Primary を白背景上で使用し 5.12:1 / 4.61:1 を確保。タグ文字は Tint 背景 + Dark 文字で 8.3:1 以上を実現。 ドライブ中のチラ見や振動時の読み取りにも耐えられる可読性を優先しています。
