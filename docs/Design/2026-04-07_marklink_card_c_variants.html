パターンC バリエーション提案

タイムライン連結型の深掘り。距離ラベルをカード外（タイムライン縦線横）に配置する4つのバリエーションを提案します。

作成日: 2026-04-07 役割: designer 参照元: 2026-04-07_marklink_card_design_proposal.html

設計制約（今回のフィードバック）

距離情報はカード内に表示しない。 タイムラインの「線」部分に紐づく情報として、カード外（縦線の横・線の中央バッジ等）に配置すること。

バリエーション概要

Mark(地点A) → Link(区間: 50km) → Mark(地点B) の3要素を各バリエーションで描画しています。距離ラベルはすべてカード外に配置。

C-1 サイドラベル型 — 縦線横に小テキスト浮かせ

タイムライン縦線の左側に距離テキストを小さく浮かせるシンプルな配置。縦線は細め（2px）で統一し、ドットは Mark=大円・Link=小菱形で形状差別化。カードは角丸控えめでフラット寄りの印象。

縦書き距離テキスト: 縦線左横に配置

特徴・デザインポイント

  • 距離テキスト「50 km」を縦書きで縦線の左側に配置。タイムライン帯域内に収まり、カードへの干渉ゼロ。
  • Mark ドット=大円(18dp)、Link ドット=菱形(12dp回転)で形状を明確に差別化。色＋形の二重識別。
  • 縦線は細め(2px)グラデーションで Mark→Link の色変化を滑らかに表現。
  • Mark カード: 白背景・上ボーダー3px・角丸8px。Link カード: 薄緑背景・1pxボーダー・角丸6px。
  • 最もシンプルな構造。実装コストが低く、フォールバックとしても機能する。

C-2 中央フローティングバッジ型 推奨

Mark→Mark を結ぶ縦線の中央に距離バッジを「浮かせて」配置。縦線はLink区間を太く(4px)して視覚的な重みを持たせ、バッジは小さな丸み角のラベルで存在感を与える。カードは角丸を強め（Mark:16px / Link:12px）にしてやわらかい印象。

距離バッジ: Link区間縦線の中央に浮かせ

特徴・デザインポイント

  • 距離バッジ「50 km」を Emerald 色の小ラベルで Link 縦線中央に浮かせる。視認性が高く「線の情報」であることが直感的に伝わる。
  • Link 区間の縦線を太く(6px・グラデーション)することで「区間の重み」を表現。Mark 区間の細い縦線との対比で情報密度が明確。
  • カードの角丸を強め(Mark:16px / Link:12px)にすることでモバイルアプリらしい親しみやすさを演出。
  • 距離バッジに小さな影(box-shadow)を添え、縦線と重なった際でも読みやすく浮き上がって見える。
  • 距離と区間が視覚的に一体化しており「50km 走った」というナラティブが伝わりやすい。

C-3 インライン区切り型 — 距離を線間のセパレータに

Mark と Link カードの間に「距離セパレータ行」を独立したUIとして挿入するアプローチ。縦線を点線にすることで「移動中=不連続」な雰囲気を演出。距離は縦線上の独立コンポーネントとして扱う。

点線縦線 + 距離セパレータ行

特徴・デザインポイント

  • 距離情報を「セパレータ行」として独立したUI要素に昇格。「--- 50 km ---」表記で「移動中の距離」であることを明示的に示す。
  • 縦線を点線(dashed)にすることで「区間=移動中=非静止」の雰囲気を視覚的に表現。Mark の実線とのコントラストが区別を強調。
  • Mark ドットを「枠のみ（塗りなし）」にすることで、Linkとの対比をより強くしつつ、落ち着いた雰囲気を演出。
  • Mark カードを左ボーダー型(border-left:4px)に変更。上ボーダー型よりも「縦線との連続感」が高い。
  • 距離セパレータの破線ボーダーが点線縦線と視覚的に呼応し、デザインに統一感をもたらす。

注意点: 距離セパレータ行がリストに挿入されることで、スクロール時の視点の流れが一度止まる効果がある。密度の高いタイムラインでは情報量が増えすぎる可能性があり、Mark/Link 数が多い場合の検討が必要。

C-4 ルート地図型 — 縦線を太く・距離を沿道テキストで

縦線を道路のように太くし、その右脇に小さな距離テキストを水平で添えるアプローチ。「道路地図の縮小版」のような視覚体験を目指す。カードは影を強め(shadow-lg)に、輪郭をはっきりさせたデザイン。Mark と Link は縦線の太さで差別化（Mark=細・Link=太）。

極太Link縦線 + 沿道テキスト配置

特徴・デザインポイント

  • Link 区間縦線を極太(11px)にして「道路」を表現。距離テキストを縦線の右脇に水平で添えることで「沿道の距離標識」のような体験を演出。
  • Mark ドットを「外枠のみ＋内側の小円」の二重構造にすることで、地図ピンのようなリッチな印象を付与。
  • Mark カードの左端に極細カラーライン(box-shadow left)を追加し、タイムライン縦線との視覚的つながりを強調。
  • 縦線の太さで Mark区間(3px)とLink区間(11px)を劇的に差別化。色だけでなく太さでも文脈が伝わる。
  • 「道路地図」メタファーが最も強いデザイン。MichiMarkのコンセプトを最も視覚的に体現する。

推奨バリエーションと理由

4つのバリエーションを比較し、総合評価から推奨を判定します。

C-1: サイドラベル型

  • 縦書き距離: 視認性やや低
  • 実装コスト: 低
  • 直感性: 中
  • 菱形ドット: 新鮮な差別化
RECOMMENDED

C-2: 中央バッジ型

  • 距離バッジが最も直感的
  • 太い縦線と距離が一体化
  • 実装コスト: 中
  • 可読性: 高

C-3: セパレータ型

  • 距離の独立性が最も高い
  • 点線が移動感を演出
  • 実装コスト: 中
  • 密なリストでは重い可能性

C-4: ルート地図型

  • 道路メタファーが最強
  • 太い縦線で画面幅を消費
  • 実装コスト: 中〜高
  • コンテンツ幅に注意
✅

推奨: C-2（中央フローティングバッジ型）

距離情報の「カード外配置」制約を最も自然に満たしながら、可読性・視覚的直感性・実装バランスが優れているのが C-2 です。

  • 距離バッジが Link 縦線の中央に浮かびつつカードに干渉しないため、「線の情報=距離」というメンタルモデルが最も伝わりやすい
  • Emerald グリーンの太い縦線(6px)と距離バッジが色・位置ともに一体化し、「SA足柄→50km走行→大井松田」のナラティブがひと目で読み取れる
  • カード角丸を強め(Mark:16px / Link:12px)にすることでモバイルネイティブらしいやわらかい印象を実現
  • 縦書き(C-1)・セパレータ行挿入(C-3)・極太縦線(C-4)と比べ、実装の複雑度と視覚的効果のバランスが最良
