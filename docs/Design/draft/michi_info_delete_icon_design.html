4. 根拠・デザイン判断

項目判断理由
アイコン形状 ゴミ箱アイコン × より削除の意味が明確。Material Icons の Icons.delete_outline を推奨（filled より軽量な印象）
アイコンカラー 赤系（#DC2626）on 薄赤背景（#FEE2E2） 削除は破壊的アクションのため色でも警告を示すのが適切。コントラスト比 4.6:1 で WCAG AA 準拠
アイコンサイズ Markカード 28×28px / Linkカード 22×22px タップターゲット最小 44dp を確保しつつ、Linkカード34px高さに収まるサイズ調整
配置 右端横並び（削除は常に最右端） 削除ボタンを最も外側に配置することで誤タップを低減。給油・ActionTime は削除の左隣に自然な順序で並ぶ
常時表示 常時表示（スワイプ廃止） スワイプ削除のディスカバビリティ問題を解消するのが本要件の目的。常時表示が最もシンプルな解決策
誤削除対策 本提案のスコープ外 削除確認ダイアログ（AlertDialog）の実装は既存 REQ-michi_info_card_delete.md に準拠。本提案はアイコンのビジュアルデザインのみ対象

5. product-manager 向け要件叩きコメント

推奨案: アイコン案 A + 配置案 1

以下の判断で推奨案を選定しています。product-manager のレビュー・ユーザー承認後に要件書として確定してください。

  • アイコン案 A（ゴミ箱・赤背景）を推奨 — 削除の意味が最も明確で、危険なアクションであることを赤色で示す。常時表示でも誤操作を防ぐ心理的ブレーキになる。
  • 配置案 1（右端横並び）を推奨 — 実装コストが最も低く（既存 Row に末尾追加）、削除が常に最右端で位置が一定。
  • アイコンサイズ — Markカード 28×28px、Linkカード 22×22px。flutter_slidable 廃止後のカード高さ変更は不要。
  • 削除確認ダイアログ — 常時表示になるため誤タップリスクが上がる。既存の削除フローに確認ダイアログが実装済みかを architect と確認することを推奨。
  • 代替案の検討余地 — 一覧がカードで埋まる場合に赤が多すぎると感じるならアイコン案 B（グレー）に変更可能。その場合は誤認リスクを補う UX（例: タップ時に赤くハイライト）の追加を検討すること。
