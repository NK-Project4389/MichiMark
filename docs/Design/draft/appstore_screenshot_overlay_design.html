設計根拠

各デザイン決定の根拠・Apple App Store ガイドライン準拠ポイントを記載。

レイアウト構成の根拠

要素 比率（実寸） 根拠・ガイドライン準拠
上部（キャッチコピーエリア） 約 28%（〜803px） Apple ガイドライン「テキストが全体の 30% 以下」を遵守。 大きすぎると App Store 審査での表示縮小時に潰れるため、28% に設定。 アプリ名バッジ（uppercase・字間広め）でブランドを印象付け。
中央（スクリーンショットエリア） 約 55%（〜1577px） 実機スクリーンショット＋iPhoneモック枠を配置。 モック枠があることで「アプリの画面」と直感的に認識させる。 Dynamic Island を模したデザインで iOS 17+ を想起させる。
下部（サブテキストエリア） 約 17%（〜488px） 補足情報を白 82% の薄めテキストで表示。 Apple が推奨する「スクリーンショット下部の空きスペース活用」に準拠。 2行以内に収めることで、App Store ギャラリー縮小時にも視認可能。

5枚の役割分担と優先度

枚 画面 役割 グラデーション 優先度
1枚目 タイムライン一覧 「何のアプリか」を最速で伝える。ダウンロード前に最初に目に入る1枚。ドライブ記録の全体感を見せる Primary（深緑 → Teal） 最重要
2枚目 地点・区間 詳細 MichiMark 独自の「地点 + 区間」構造を説明。他のドライブアプリとの差別化ポイント Deep（暗いTeal） 高
3枚目 集計タブ 費用・燃費の自動集計を訴求。「記録が活きる」体験を見せる Warm（Teal → Emerald → 明るい緑） 高
4枚目 支払い・精算 グループ旅行・割り勘ユーザーへの訴求。「旅費精算まで完結」というバリュー Ocean（Teal → 深い緑） 中
5枚目 地点追加・トピック 入力の簡単さとトピックの多様性。カジュアルユーザーへの「使いやすさ」訴求 Night（深いネイビー） 中

キャッチコピーの設計方針

枚 コピー 設計意図
1 走ったルートを、タイムラインで残す。 動詞（走る・残す）で「行動の記録」を明示。「タイムライン」を強調語に。読点で溜めを作り視線誘導
2 地点・区間・費用、まとめて記録。 3要素を列挙することで情報密度の高さを示す。「まとめて」で手間の少なさを暗示
3 旅費も燃費も、自動で集計。 「も〜も」で対比構造。「自動で」で手軽さ・利便性を前面に。燃費は他アプリにないUSP
4 割り勘精算、旅の帰りに終わる。 「旅の帰りに」というシチュエーション設定で使用場面をイメージさせる。精算の即時性を訴求
5 ドライブ・仕事・旅行、用途別に記録。 3ターゲット（ドライバー・ワーカー・旅行者）を並列列挙。検索キーワードにも合致

実制作時の注意事項（Figma / 実機キャプチャ差し替え時）

項目 内容 ステータス
実機スクリーンショット docs/AppStore/screenshots/ の実機キャプチャ5枚を使用。最新ビルドで再撮影する場合は同ディレクトリのファイルを差し替えること 対応済み
出力サイズ 1320 × 2868 px（iPhone 6.9インチ最大解像度）。Retina対応で @3x 出力推奨 要確認
フォント 日本語: 実制作では「Hiragino Sans W8」または「Noto Sans JP Black」を使用。本レポートは近似フォントで代用 要確認
Apple ガイドライン テキスト占有率 28%（30% 以下）・コントラスト比 WCAG AA 以上を確認済み 準拠
Dynamic Island iPhone 15 Pro / 16 系の実機キャプチャには Dynamic Island が写る。モック枠のノッチ形状と一致させること 要確認
英語版対応 App Store の多言語対応を行う場合は英語版キャッチコピーを別途作成すること 将来対応

# デザイン要件の叩き: App Store スクリーンショット オーバーレイ ## 概要 MichiMark の App Store スクリーンショット（5枚）に、 テーマカラー（Teal/Emerald）グラデーション背景 + キャッチコピー + サブテキストを 重畳したオーバーレイ画像を作成する。 現状のスクリーンショット（キャプチャのみ）から、 アプリの価値が一目で伝わるビジュアルへ刷新する。 ## 変更項目 - [ ] 1枚目: タイムライン一覧 オーバーレイ作成（Primary Grad） - [ ] 2枚目: 地点・区間 詳細 オーバーレイ作成（Deep Grad） - [ ] 3枚目: 集計タブ オーバーレイ作成（Warm Grad） - [ ] 4枚目: 支払い・精算 オーバーレイ作成（Ocean Grad） - [ ] 5枚目: 地点追加・トピック オーバーレイ作成（Night Grad） - [ ] 各スクリーンショット: 実機キャプチャ取得（最新ビルドで再撮影） - [ ] 出力サイズ確認: 1320 × 2868 px @3x - [ ] App Store Connect へのアップロード・確認 ## デザイン仕様（各オーバーレイ共通） - 背景: 枚ごとの指定グラデーション（Primary / Deep / Warm / Ocean / Night） - 上部エリア: 高さ 28%（キャッチコピー + アプリ名バッジ） - 中央エリア: 高さ 55%（実機スクリーンショット + iPhone モック枠） - 下部エリア: 高さ 17%（サブテキスト 2行以内） - キャッチコピー: 82px / 太さ800 / 白 / 強調語は Emerald 200（#A7F3D0） - サブテキスト: 46px / 太さ500 / 白 82% - フォント: Hiragino Sans W8（日本語） ## 参照デザインレポート docs/Design/draft/appstore_screenshot_overlay_design.html ## 優先度 高（App Store 公開直後・CVR 改善施策として即時対応推奨） ## 備考 - 英語版コピーは将来対応（現時点は日本語のみ） - Figma ファイルへの展開は marketer / designer と連携して進める
