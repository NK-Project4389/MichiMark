ピンの形そのものを「道路＋センターライン」で構成。ピンのシルエットが道のシンボルを内包する最もシンプルな統合案。

背景#1B6D8E ソリッド

メジャーピン両脇にルーラー目盛り（サブ要素）

RATIONALE

デザイン根拠

各デザイン判断の背景と選択基準を記録する。

アイコンからテキストを除去した理由

iOS ホーム画面の最小表示サイズ（20x20px程度）でテキストは完全に潰れる。App Store のアイコンはシンボルのみで識別できるべき（Apple HIG 準拠）。ブランド名はウィジェット・スプラッシュ・ストアページで補完する。

グラデーションを極力排除した理由

ビジネスユーザー（ドライバー・現場作業員）に向けたクール・信頼感重視のデザイン方針に沿い、ソリッドカラーを基調とする。光沢感はハイライトシェイプ（白透過楕円）で最小限に留め、後から任意で除去できる構造にした。

ピンをソリッド（白塗り）にした理由

現行はアウトラインピンだが、小サイズでアウトラインは視認性が落ちる。ソリッド白は背景の濃いブルーとの高コントラストを維持し（コントラスト比 > 7:1）、あらゆるサイズで認識できる。

アンバーアクセントを採用した理由

全面ブルー系だとピンと道路が同系色に埋もれやすい。アンバー（#E8A020）は補色に近い位置にあり、「計測ポイント」や「マーク済み地点」を視覚的に際立たせる。アイコンのメインカラーとしては使わず、アクセントドットに限定して使用。

道路の向き（横断 vs 縦）の選択

現行は縦方向（下に道路）。スプラッシュでは「ミチが横に横断する」というユーザーのコンセプトを忠実に再現し、水平方向の道路を採用。アイコン（正方形）では構図の柔軟性が高い縦方向のピン内道路（Plan A）が安定する。

WCAG コントラスト比確認

  • 白 on #1B6D8E: 約 4.9:1（AA 合格）
  • 白 on #0F4A62: 約 8.2:1（AAA 合格）
  • 白 on #2589AD: 約 3.8:1（AA Large 合格）
  • アンバー #E8A020 on Deep: 約 5.6:1（AA 合格）

NEXT STEPS

提案後の進め方

このレポートは designer によるドラフト。採用案の確定後、product-manager が要件書に整形する。

  1. ユーザーが アイコン案（A / B / C）とスプラッシュ案（S1） を確認・フィードバック
  2. 採用案確定 → product-manager が 要件書（docs/Requirements/） に整形
  3. 要件書をもとに Figma データ作成（SVGをFigmaにインポート → 実サイズ 1024x1024px でトレース）
  4. Figma から 1024x1024 PNG エクスポート → flutter_launcher_icons / flutter_native_splash で各解像度を自動生成
  5. スプラッシュアニメーションは 別フェーズ（要件書・Spec分離）で実装
