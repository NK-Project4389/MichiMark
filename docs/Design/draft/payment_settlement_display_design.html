T-303 / UI-9 Design Proposal

「支払いごとの精算」誤認防止デザイン変更

EventDetail 概要タブの精算表示が「タップできる」ように見える問題を解消するための 静的表示デザイン提案。A案・B案・C案の3案を比較提示する。

作成日: 2026-04-13  |  役割: designer  |  対象画面: EventDetail > 概要タブ
Problem
現状の問題
現在の「支払いごとの精算」は Flutter の Card ウィジェットで描画されており、 shadow・elevation・角丸が付与された「タップ誘引デザイン」になっている。 実際はタップ不可（ビジネスロジックなし・集計表示のみ）にもかかわらず、 ユーザーが誤ってタップ可能だと判断する。
問題点の整理
  • Card の elevation による浮き上がりがインタラクティブ要素のシグニファイアになっている
  • 同じセクション内の他の行（経費合計・メンバー別コスト・収支バランス）はフラットな Row 表示なのに、精算カードだけ Card になっており視覚的不統一
  • タップしても反応がないため UX 上の違和感・期待外れが発生する
  • リップルエフェクトはないが Card の外観だけでタップを誘引している

Rationale
B案を推奨する根拠

シグニファイアの正確な使用

elevation（影・浮き上がり）はインタラクティブな要素のシグニファイアとしてユーザーに広く認識されている。 「左ボーダー＋ティント背景」はスタイルシートやドキュメントビューアで広く使われる 「引用・参照ブロック」のパターンであり、タップ不可・読み取り専用の印象を自然に与える。

WCAG AA 基準の充足

B案のティント背景（#EAF5FB）とテキスト（#1A1A2E）のコントラスト比は約 14:1 であり、 WCAG AA 基準（4.5:1）を大きく上回る。 左ボーダー色（#2B7A9B）は装飾のみで情報伝達には使わないため基準外。

伝票が複数あるときの視認性

支払いが3件・4件と増えたとき、A案のフラット区切り線は読み流しやすく伝票境界が曖昧になる。 B案のティントブロックは背景色のまとまりで1伝票 = 1ブロックを直感的に区切る。

MichiMark らしさとの統一

ティント色（#EAF5FB）と左ボーダー（#2B7A9B）はすでに他の画面で使用されている MichiMark ブランドカラーからの派生であり、新しいカラーを追加しない。 デザイン全体の統一感を損なわない。

Draft / T-303

デザイン要件の叩き: 支払いごとの精算 誤認防止デザイン変更

概要

EventDetail 概要タブ「支払いごとの精算」セクションの表示UIを、 タップ誘引のある Card（elevation あり）から、 静的な「背景色ティント＋左ボーダー」スタイルに変更する。 表示する情報・ロジックへの変更は一切行わない。

対象画面・コンポーネント

  • 画面: EventDetail > 概要タブ（travel_expense_overview_view.dart）
  • 対象クラス: _PerPaymentSettlementCard

変更項目

  • _PerPaymentSettlementCard の Card ウィジェットを撤去する
  • 代替として背景色 #EAF5FB（brand-teal-tint2）・左ボーダー #2B7A9B 幅3pxのコンテナで包む
  • 角丸は borderRadius: 8px を維持する（視覚的な角張り感をなくすため）
  • elevation・shadow は完全に除去する（0）
  • 内側 padding は現行の 12px から 8px 10px に微調整する（コンパクト化）
  • タップイベント（InkWell・GestureDetector）は追加しない
  • クラス名を _PerPaymentSettlementCard から _PerPaymentSettlementBlock にリネームする（Card でないことを明示）

変更しないもの

  • 表示する情報（伝票タイトル・金額・精算行）の構造・内容
  • Projection・Adapter・BLoC レイヤーのロジック
  • 他セクション（経費合計・メンバー別コスト・収支バランス）のスタイル
  • _SettlementLineRow の文字色（payer: error色、receiver: green）

採用案

B案（背景色ティント＋左ボーダースタイル）を採用する。詳細は参照デザインレポートを参照。

参照デザインレポート

docs/Design/draft/payment_settlement_display_design.html

受け入れ基準

  • 「支払いごとの精算」エリアにホバー・タップ時のリップルが発生しない
  • elevation（影）が視認されない
  • 各伝票ブロックが背景色で視覚的に区切られている
  • 表示される金額・名前・精算行の情報が変更前と一致する
  • WCAG AA コントラスト比 4.5:1 以上を満たす（ティント背景＋テキスト）
