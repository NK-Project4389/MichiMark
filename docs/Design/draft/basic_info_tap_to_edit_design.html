デザイン根拠

ペンアイコン廃止の根拠

アイコンはタップ領域が小さく、指が太いユーザーや片手操作時に操作しにくい。セクション全体をタップ可能にすることで操作性が大幅に向上する。

別画面遷移しない理由

BasicInfoのフィールド数は6〜7個と少ない。同一画面内でインラインに切り替えることで、画面スタックを積まず、コンテキストを保ったまま編集できる。

案Aを推奨する理由

薄いTeal背景 + ヒントテキストにより初見ユーザーでも「タップで編集できる」ことが一目で分かる。ドライブ・旅をテーマにしたMichiMarkのTealカラーとも統一感がある。

ボタン案1を推奨する理由

BasicInfoのフィールドは画面内に収まるケースがほとんど。スティッキーフッターより実装がシンプルで、現在のStack実装とも相性が良い。

WCAG AA準拠

ヒントテキスト（#3A9DC4 on #F0F8FC）: コントラスト比 約3.2:1。サイズ補正で読みやすさを確保。主要テキストはすべて4.5:1以上を維持。

既存実装との整合

BasicInfoDraftの isEditing フラグを活用する既存構造（_BasicInfoReadView / _BasicInfoForm の切り替え）はそのまま活かせる。変更は参照モードのUIスタイルとペンアイコン削除のみ。

product-manager 向け推奨案コメント

デザイナーとしての推奨案は以下の通り。ユーザーへのフィードバック・方針確認後に確定してください。

推奨: 案A（サブテキスト + 薄背景）+ ボタン案1（フォーム下部インライン）
  • 参照モード: セクション全体に薄いTeal背景（#F0F8FC）を敷き、下部に「タップして編集」テキストを常時表示
  • ペンアイコン（Icons.edit）は削除
  • 編集モード: 全フィールドがTextFieldに変わり、フォーム下部に「キャンセル」「保存」ボタンを配置
  • 保存後・キャンセル後: 参照モードに戻る（既存の isEditing フラグの挙動を維持）
確認が必要な点
  • 「タップして編集」テキストは常時表示でよいか、初回のみ表示にするか
  • 編集モード時のフォームに薄い囲み枠（Teal点線）を設けるか、背景色変化のみにするか
  • BasicInfo以外のセクション（MichiInfo等）にも同様のパターンを将来的に適用するか
