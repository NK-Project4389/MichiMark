デザイン根拠

A案（シェブロン + ホバー背景）を推奨する理由。

推奨案: A案 — シェブロン + ホバー背景 + ヒントバナー

iOSの標準リストUIで広く使われているシェブロン（›）パターンは「タップすると別画面へ進む」という ユーザーの既存メンタルモデルと一致する。MichiMarkのBasicInfo参照モードでは 各フィールドをタップすると編集フォームへ遷移するため、このパターンが最も自然に適合する。

加えて、初回表示時にのみ「タップして編集できます」という小さなヒントバナーを画面上部に表示することで、 初めて使うユーザーへの気づきを補完できる（常時表示ではなく初回のみが理想）。

iOSメンタルモデルへの適合

›（シェブロン）は「タップで詳細・編集へ進む」をネイティブが示す標準パターン。ユーザーが学習コスト0で理解できる。

WCAG AAコントラスト確認

ラベルテキスト（#8A94A0 on #FFFFFF）: コントラスト比 約3.5:1。値テキスト（#1A1A2E on #FFFFFF）: 約16:1。ホバー背景（#E8F4F8）でも値テキストのコントラストは維持される。

実装コストの最小化

現状の _ReadRow Widgetに「末尾にシェブロンアイコンを追加」「行全体をGestureDetectorでラップ」するだけで実現可能。レイアウト変更が最小限。

シンプル・ミニマルの維持

B案のTeal背景は視覚ノイズが大きい。A案は通常状態でシェブロンのみ追加するため、情報密度を高めずにアフォーダンスを付与できる。

未設定値への一貫適用

「未設定」「未選択」の行にもシェブロンを表示することで、設定を促すCTAとしても機能する。視覚的に設定済み行と区別されるため、どこを編集すべきかが一目でわかる。

MichiMarkブランドへの適合

シェブロンカラーは中立グレー（#ADB5BD）を使用し、Tealはホバー背景とヒントバナーに限定。ブランドカラーの使いすぎを避け、MichiMarkらしいシンプルな外観を維持。

要件の叩き（product-manager確認用）

# デザイン要件の叩き: BasicInfo 参照タップ編集UI（UI-2）

## 概要
BasicInfoセクションの参照モードにおいて、編集アイコン（ペンアイコン）を廃止し、
各フィールド行をタップすることで編集フォームへ遷移するUIに変更する。
タップ可能であることを示すビジュアル表現として「A案：シェブロン + ホバー背景 + ヒントバナー」を推奨する。

## デザイン推奨案: A案（シェブロン + ホバー背景 + ヒントバナー）

## 変更項目
- [ ] _BasicInfoReadView の右上 IconButton（Icons.edit）を撤去
- [ ] _ReadRow の末尾にシェブロンアイコン（Icons.chevron_right）を追加
- [ ] _ReadRow 全体をタップ可能にする（InkWell または GestureDetector でラップ）
- [ ] タップ時に BasicInfoEditModeEntered を発行する（既存イベントを流用）
- [ ] 初回表示時にのみ「タップして編集できます」のヒントバナーを上部に表示（任意・初回のみ）
  ※ ヒントバナーの初回判定方法はarchitectが設計する

## タップ可能ビジュアル仕様（A案）
- シェブロンアイコン: Icons.chevron_right、サイズ 18、色 #ADB5BD（グレー）
- タップ時ハイライト: InkWell のデフォルトリップル or 背景色 #E8F4F8（Teal Pale）
- ヒントバナー（任意）:
  - テキスト: 「タップして編集できます」
  - 背景色: #E8F4F8 / テキスト色: #2B7A9B
  - 表示条件: 初回表示のみ（SharedPreferences等で管理、設計はarchitectに委ねる）
