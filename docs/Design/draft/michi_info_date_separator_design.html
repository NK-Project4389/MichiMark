Designer Report — UI-23

MichiInfo 日付区切り表示

MarkiLinkカードの間に日付区切りを挿入し、時系列の把握を補助するデザイン提案。既存の道路帯タイムラインと共存する最小介入の実装案。

2026-04-17 対象: MichiInfo タイムライン画面 全トピック共通
01 Overview

設計方針

道路帯タイムライン（UI-14確定案）の縦ラインを切断せず、日付区切りを「道路上の日付標識」として挿入する。

前提: 道路帯（グレー帯 + 白破線）はCanvasで全体を通して描画する（_MichiTimelineCanvas）。日付区切りはFlutterウィジェットとして各カードの間に配置し、道路帯レイヤーは切断しない。

表示条件: 前のカードと日付が変わる場合のみ表示。同日が続く場合は非表示。リストの先頭アイテムの前にも表示する（その日の最初のカードとして視覚的に分かりやすくするため）。

位置づけ: 区切りはSliverListのアイテムとして挿入（インジケーターと同様の仕組み）。InsertMode中は非表示（Y座標ずれ防止の既存ロジックに合わせる）。

07 Rationale

デザイン根拠

このデザインを選択した理由。

道路帯を切断しない

区切りはFlutterウィジェット（Widget）として挿入し、Canvasレイヤーはそのまま。道路帯のグレー帯はアイテム全体を貫通する連続した縦線として描画され、区切りを「跨ぐ」形になる。

全トピック共通で使える無彩色

推奨案Aのスレートグレー系は、訪問作業・ドライブ・その他トピック全てに適用可能。Teal（Mark色）やEmerald（Link色）と干渉しない。代替案BのTeal系はドライブトピック特有の印象を強める。

シンプルで情報量を増やさない

区切りは「存在を知らせる」だけでよい。アイコン・背景色変化など余計な要素は加えない。横線＋日付バッジの最小構成で日付の変わり目を伝える。

Y座標計算への影響を局所化

区切り挿入によるY座標ずれは_buildTimelineDataで区切り高さ(48px)を加算することで対処。InsertMode中は区切りを非表示にしてInsertModeの既存ロジックと競合しないようにする。

08 Implementation Note

実装メモ

architect / flutter-dev への申し送り事項。

SliverList の childCount と区切り挿入ロジック

現在の非InsertModeのSliverChildBuilderDelegateは childCount: items.length でインデックスをアイテムに直接マッピングしている。

日付区切りを挿入する場合、InsertModeと同様に「区切り込みのインデックス列」を事前計算するか、別のウィジェットリストを生成するアプローチを採用する。実装方式はarchitectが判断する。

_buildTimelineData への影響: 区切りコンポーネントの高さ（48px）をY座標計算に加算する必要がある。区切りが入る位置（何件目の後か）を特定し、それ以降のアイテムのY座標を48px分ずらす。

日付比較: markLinkDate（DateTime）の year/month/day 部分のみ比較。時刻は無視する。

InsertMode中の非表示: isInsertMode == true のとき区切りウィジェットを表示しない（現在の道路帯非表示ロジックと同様に扱う）。
