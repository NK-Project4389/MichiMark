各画面の刷新後デザインを実寸近似でプレビュー
□
OutlineInputBorder
テキスト入力は常に枠線付き。通常は Teal Light (#B2DFDB)、フォーカス時は Teal (#4ECDC4) + 薄いグロー。
▸
選択行の明示
選択可能行は背景 #F0FAFA + ボーダー #3AADA5 + 塗りつぶし丸矢印アイコン。テキスト入力と明確に区別。
⎯
Divider必須
各フォーム項目間に #E5F0EE の1px Divider。セクション境界は #B2DFDB の強めDivider。
⬬
ボタン統一
保存 = ElevatedButton（Teal Dark背景・白テキスト）。キャンセル = TextButton（グレーテキスト・背景なし）。
▶
タグ3層構造
登録済みタグ（選択解除可チップ）/ レコメンド（追加可チップ）/ 新規入力フィールドを明確に分離。
▲
FABデザイン
MichiInfo の FAB は Teal Dark 背景・白アイコン。ラベルあり拡張FABを基本とし、スクロール時はアイコンのみに縮小。

デザイン根拠
各変更の理由と原則との整合性
OutlineInputBorder の採用
Flutter Material3 のデフォルトは UnderlineInputBorder であり、境界が下線のみで視認しにくい。iOS ネイティブ設定画面に近い印象を出すには枠線ありの入力フィールドが適切。通常時は Teal Light (#B2DFDB) の薄い枠でシンプルに抑え、フォーカス時に Teal (#4ECDC4) + グロー (opacity 0.15) に変化させることでフィードバックを明確にする。
選択行の視覚的差別化
テキスト入力行と選択行が同じスタイルでは「タップして選ぶ」か「ここに打ち込む」かを判別できない。選択行は背景を #F0FAFA（Teal Tint2）+ ボーダー #3AADA5 に設定し、矢印も Teal Dark (#2D6A6A) の塗りつぶし丸アイコンに強化する。これにより「選択可能な行」が一目でわかる。WCAG AA のコントラスト比（背景 #F0FAFA に対してラベル #3E5A5A は約 6.2:1）を満たす。
タグセクションの3層分離
登録済みタグ（選択解除可）/ レコメンドタグ（追加可）/ 新規入力 フィールドの3層を、背景色と見出しラベルで分離する。登録済みは白背景・Teal枠チップ、レコメンドは #F9FCFC 背景・グレー枠チップ、入力フィールドは通常のテキスト入力。ユーザーが「今どの操作ゾーンにいるか」を視覚的に理解できる。
ボタン統一 (ElevatedButton / TextButton)
保存ボタンは ElevatedButton（Teal Dark 背景・白テキスト・角丸 12px）で主アクションを強調。キャンセルは TextButton（グレーテキスト・背景なし）で副次アクションを示す。これにより画面内のアクション優先度が一目で伝わる。ナビゲーションバー内にも同様のテキストアクション（左: キャンセル / 右: 保存）を配置し、スクロールなしでも操作できる。
Divider の必須化
項目間に #E5F0EE の1px Divider を設けることで、スキャン時の視線の流れを整理する。セクション（燃料情報・タグ等）の境界は #B2DFDB の強めの Divider で更に区別する。余白だけでは項目数が増えた際に境界が曖昧になるため Divider は必須。
FAB のスタイル統一
MichiInfo の FAB は Teal Dark (#2D6A6A) 背景・白アイコン・角丸 16px の拡張 FAB を基本とする。スクロール時は円形 FAB に縮小してコンテンツ領域を確保。タップ時は背景色を #245858（ダーク方向に 5%シフト）にして視覚フィードバックを与える。ブランドカラーと一致し、かつカード群（タイムライン）と視覚的に区別される。
