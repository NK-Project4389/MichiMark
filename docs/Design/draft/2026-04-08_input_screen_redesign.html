デザイン根拠
各変更の理由と原則との整合性（v2確定版）
ボーダーレス入力フィールドの採用
OutlineInputBorder はフォーム感が強く、MichiMark が目指すシンプル・ミニマルなドライブ記録アプリの雰囲気と合わないとユーザーが判断。border: InputBorder.none + Divider（height: 1）の組み合わせは、iOS の設定画面・メモアプリに近い自然なレイアウトを実現する。枠線がない分、コンテンツそのものが前面に出てスタイリッシュな印象になる。
選択行のシンプル化（背景色・枠線廃止）
v1提案の背景色 #F0FAFA + 枠線 #3AADA5 は「色で選択行を区別する」意図があったが、ユーザーが「微妙」と判断。入力行との差異が過剰で、全体的なシンプルさを損なう。chevron_right アイコン（›）のみによる識別は iOS のデザインパターンとして広く浸透しており、ユーザーが直感的に「タップして選ぶ」と理解できる。WCAG AA のコントラスト要件はテキスト色と背景白の組み合わせで引き続き満たす。
タグセクションのラベル名変更
「レコメンド」は何をレコメンドするのかが曖昧。「最近使用したタグ」とすることで、ユーザーが「過去に使ったタグが出ている」と即座に理解できる。「タグを追加」も「新しいタグを追加」とすることで「既存タグを追加するのか、新規作成するのか」の曖昧さを解消する。実装コードでも hintText: '新しいタグを追加' として反映済み。
Divider 区切りとインライン行レイアウトの維持
height: 1 の Divider による行区切りは、ボーダーレスのテキストフィールドと組み合わせることで項目境界を明示する唯一の視覚要素となる。ラベル幅 120px（bodySmall / onSurfaceVariant）+ Expanded フィールドのインライン行は全画面統一レイアウトとして維持する。これにより複数画面（BasicInfo / MarkDetail / LinkDetail / PaymentDetail）でスキャン性が統一される。
拡張 FAB の維持
アイコン＋「追加」テキストの拡張 FAB は「何ができるか」を明示する点でシンプルな丸ボタンより優れている。Teal Dark (#2D6A6A) 背景・白テキスト・角丸 16px のスタイルはブランドカラーと一致し、タイムライン画面上で視認性が高い。ユーザーからもそのまま採用の承認を得ている。
