# デザイン要件の叩き: カスタム数値キーパッド

## 概要

MichiMark の全数値入力フィールド（`NumericInputRow` 使用箇所）に対して、
デバイス標準キーボード（`TextInputType.number`）を廃止し、
専用のカスタム数値キーパッド（カスタムIME）に置き換える。

---

## 背景・目的

- 標準キーボードはOS・機種によりレイアウトが異なり、「00」キー・演算子・Draft表示などを統一的に提供できない
- ドライブ記録ユースケード（燃費計算、給油金額の単価×量など）では簡易計算ニーズが存在する
- 入力中に変更前の値（Draft original）を参照できると誤入力訂正が容易になる

---

## 対象ウィジェット・使用箇所

- `NumericInputRow` ウィジェット
- BasicInfo（燃費・ガソリン単価）
- FuelDetail
- PaymentDetail
- MarkDetail
- LinkDetail

---

## 変更項目

### Phase 1（優先度: 高）
- [ ] `NumericInputRow` の `TextField` + システムキーボードをカスタムキーパッドに置き換え
- [ ] キーパッドをBottomSheet（Overlay）方式で表示
- [ ] キー構成: 0〜9, 00, . (小数点), C（クリア）, ⌫（バックスペース）, 確定ボタン（全幅）
- [ ] Display エリア上部に2行: 変更前の保存値（グレー）+ 現在の入力値（大文字）
- [ ] `isDecimal = false` の場合は「.」キーを非活性グレーで表示
- [ ] `MediaQuery.of(context).size.width` 比率でキーサイズを等比スケール
- [ ] ライトモード・ダークモード対応

### Phase 2（優先度: 中）
- [ ] 演算子キー（+, −, ×, ÷）の追加
- [ ] 四則演算の中間式を Display エリア3行目に表示
- [ ] = キーで演算結果を確定し入力フィールドに反映

### Phase 3（優先度: 低・UXフィードバック次第）
- [ ] Push-up 方式への移行（システムキーボードと同様に画面コンテンツを押し上げる）

## デザイン仕様サマリー

### 採用案
**C案（5列ハイブリッド型）** を採用推奨。
詳細はデザインレポートを参照。

### キー配置（C案）
```
[C]  [7] [8] [9] [÷]
[⌫] [4] [5] [6] [×]
[.]  [1] [2] [3] [−]
[00] [0]     [+]      ← +は最後に使用した演算子（または不使用時は非表示）
[        確定         ]  ← 全幅
```

### カラー（ライトモード）
| キー種別 | 背景色 | テキスト色 |
|---|---|---|
| 数字 (0-9, 00) | #FFFFFF | #1A1A2E |
| 演算子 (+−×÷) | #D8EDEB | #2D6A6A |
| 確定 | #2D6A6A | #FFFFFF |
| Delete ⌫ | #E8EEF0 | #4A6060 |
| C クリア | #FFE5E5 | #C0392B |
| キーパッド背景 | #F2F4F4 | — |

### カラー（ダークモード）
| キー種別 | 背景色 | テキスト色 |
|---|---|---|
| 数字 | #2C3C3C | #E8F4F2 |
| 演算子 | #1A3A3A | #4ECDC4 |
| 確定 | #4ECDC4 | #1A1A2E |
| Delete | #243030 | #9BB5B3 |
| C クリア | #3A1A1A | #FF6B6B |
| キーパッド背景 | #1C2626 | — |

### サイズ仕様（375pt 基準）
| 要素 | サイズ |
|---|---|
| キー高 | 52pt |
| キー間 gap | 8pt |
| 表示エリア高 | 90pt |
| 確定ボタン高 | 52pt |
| 全体高（C案） | 約 370pt |
| スケール計算式 | sf = screenWidth / 375 |

### タイポグラフィ
| 要素 | サイズ | ウェイト |
|---|---|---|
| 現在入力値 | 32sp | Light (300) |
| 変更前値 | 11sp | Regular |
| 演算式 | 13sp | Regular |
| 数字キー | 18sp | Medium (500) |
| 確定キー | 16sp | Bold (700) |

## 非変更項目

- `NumericInputRow` の外部インタフェース（label, unit, value, isDecimal, onChanged）は変更しない
- カンマ整形・生数値文字列の受け渡しロジックは現状維持
