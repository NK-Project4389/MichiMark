MichiInfo タイムライン — 道路帯デザイン 確定案

複数案を1本に絞り込んだ確定案。タイムライン軸に沿ってグレーの道路帯を背面に描くレイヤー構成。

2026-04-15 対象: MichiInfo タイムライン画面 案A改 確定
01 Overview

シンプル化の意図

旧案A〜Cで検討していた複数のアプローチを廃止し、「道路帯を背面に敷くだけ」という最小介入の案A改に一本化する。

旧案の問題点: 案B（カード内グラデーション）・案C（アイコン変形）はコンポーネント構造の変更が大きく、既存の描画ロジックとの整合コストが高かった。

確定案の方針: タイムライン軸（x=20）に沿った縦帯を最背面に1本追加するだけ。スパン矢印・ドット・接続線・カードはすべて既存のまま流用する。変更箇所を最小化することで、デグレードリスクをほぼゼロにする。

期待効果: 「道路の上を走っている」という MichiMark らしさを、構造変更なしに視覚的に表現できる。

04 Design Spec

デザイン仕様まとめ

実装時に参照する数値・色をまとめる。

項目 値 根拠・備考
帯の中心X座標 x = 20 既存タイムライン軸と同じ。変更不要
帯幅 18 px Markドット直径12px（r=6）+ 余白3px 両側。中心±9px。ドットがはみ出さずに収まる最小幅
帯色（グレー） #888888 道路のアスファルトらしさを出しつつ、白破線・Tealドットのコントラストを確保。#777（濃）〜#999（淡）の中間値を採用
帯の端処理 radius = 4px 直線カットは無機質すぎる。大きな丸みはカード風になりタイムラインとの差異が薄れる。4pxの小さな丸みで「道路の切れ目」感を維持
白破線（色） #FFFFFF 道路センターラインの慣習（白または黄）。#888帯に対してコントラスト比約5:1でWCAG AA基準を満たす
白破線（線幅） 1.5 px 細すぎず太すぎない。帯幅18pxに対してバランスが良い
白破線（ダッシュ） on=6px / off=4px モバイル画面での視認性を考慮。間隔が広すぎると点に見える。6/4は道路標識に準拠した比率
帯の描画Y範囲 verticalLineStartRelY 〜 verticalLineEndRelY 既存のスパン矢印縦線と同じ範囲。別途計算不要
Link区間の道路帯 Emeraldオーバーレイ グレー帯の上にEmerald（#059669）半透明矩形を重ねる方式を採用。既存のEmeraldグラデーション矩形と自然に馴染み、Link区間であることが一目でわかる。帯をEmraldに置き換えるよりコード変更が少ない
Emeraldオーバーレイ 透明度 opacity = 0.22〜0.28 下層のグレー帯が透けて「道路が緑色に変わる」印象を出す。不透明だとグレーが完全に消える
05 Implementation Note

実装メモ

designer からの申し送り事項。flutter-dev 向け。

変更箇所は _MichiTimelineCanvas の描画順のみ

既存の _MichiTimelineCanvas は CustomPainter の paint() メソッド内で各要素を順番に描画している。

道路帯は 他の全描画より前（最初）に drawRect + drawLine するだけでレイヤー構成が完成する。カード・ドット・スパン矢印の描画コードは一切触らない。

Link区間オーバーレイは、既存のEmeraldグラデーション矩形と同じ Y 範囲に drawRect を追加する。この矩形も道路帯直後・他要素より前に描く。

破線は Flutter の Paint に PathEffect を使うか、または Path + dashPattern ライブラリ（path_drawing など）で実現する。仕様決定は Spec 作成時に architect が判断する。
