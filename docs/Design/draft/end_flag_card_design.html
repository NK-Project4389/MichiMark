Designer Report — F-10

EndFlag 完了カード表現

EndFlagが設定されたアクションが押されたとき、MichiInfoのMarkiLinkカードを「完了」状態として視覚表現するデザイン提案。完了状態カードの外観と、EndFlag設定UIを提案する。

2026-04-17 対象: MichiInfo カード / ActionTime 設定UI 訪問作業トピック中心
01 Overview

EndFlagとは / 設計方針

EndFlagは「このアクションが押されたらMarkのタスクを完了とみなす」フラグ。完了カードは「終わった」ことが一目で分かりつつ、情報の読み取りも可能なスタイルとする。

EndFlagの役割: ActionDomainにフラグを追加し、そのActionが記録された（ActionTimeLogが存在する）Markカードを「完了（done）」状態として扱う。例: 訪問作業トピックで「作業終了」アクションがEndFlag=trueであれば、「作業終了」が押された地点のMarkカードが完了表示になる。

完了カードの原則: (1) 未完了カードと明確に区別できる (2) 読み取り可能な情報量を維持する (3) 既存の道路帯タイムラインと共存する (4) グレーアウトしすぎてリストが暗くならない

EndFlag設定UI: ActionTime設定画面（アクションの新規作成・編集画面）において、各アクションにEndFlagをON/OFFできるトグルUIを追加する。

完了カード バリエーション — 3案比較
推奨案 A: グレーアウト + チェック推奨

カード背景をグレー化（#F9FAFB）、上辺ラインをグレー（#9CA3AF）、タイトルに打ち消し線、ドットをグレーチェックに変更。「完了」バッジをカード右上に配置。

案 B: 完了グリーン + チェック

Emerald（#10B981）で完了を表現。Linkカードと同色系になり混同リスクあり。完了したことのポジティブ感は高い。

案 C: バッジ追加のみ（最小変更）

カードの見た目はほぼ変えず、右上に「✓ 完了」バッジのみ追加。変更が最小限で実装コストは低いが、未完了との区別が弱い。

07 Rationale

デザイン根拠

推奨案Aを選んだ理由。

グレー系は「終わった」を直感的に伝える

グレーアウトはUIデザインの慣習として「非アクティブ」「完了」「無効」を表す。ユーザーはMichiMarkに限らずあらゆるアプリでこのパターンを体験済みであり、説明なしに「完了した地点」と認識できる。

案Bを選ばない理由（Emeraldとの混同）

Emerald（#10B981）はLinkカードの専用カラーとして既に確立されている。完了カードにEmeraldを使うと「Linkと同じ種別かな？」という誤認を招く。グレー系なら既存パレットと全く異なるため混同リスクがない。

情報の可読性を維持

全体を不透明0.4などにすると文字が読みにくくなる。本提案はカード背景・ボーダー・ドットをグレー化しつつ、テキストは打ち消し線付きで残すことで「完了済みだが読める」状態を維持する。

道路帯との整合性

完了ドットがグレーになることで、道路帯（グレー#888888）との視覚的な一体感が生まれる。「道路に吸い込まれた/通過した地点」というイメージとも合致し、MichiMarkらしい表現になる。

08 Implementation Note

実装メモ

architect / flutter-dev への申し送り事項。

isDone フラグを MarkLinkItemProjection に追加

MichiInfoBlocまたはProjectionビルダーで「このMarkのIDに対してEndFlag=trueのアクションログが存在するか」を判定し、MarkLinkItemProjection に isDone: bool フラグを追加する。判定ロジックはarchitectが設計する。

_MichiTimelinePainter への影響: isDone フラグをコンストラクタで受け取り、ドット色・接続線色・カード背景色・カードボーダー色・上辺ラインカラーを分岐する。既存の isFuel と同様の追加方法を想定。

_TimelineItemOverlay への影響: isDone フラグを受け取り、(1) 地点名テキストに打ち消し線 TextDecoration.lineThrough を適用 (2) 「✓ 完了」バッジウィジェットをカード右上に追加 (3) 日時・その他テキストカラーを #ADB5BD に変更する。

ActionDomain への endFlag フィールド追加: 既存の ActionDomain に endFlag: bool（デフォルト false）を追加する。DB マイグレーションが必要かどうかはarchitectが判断する。

EndFlag設定UI: ActionTime設定画面（アクション追加・編集）に CupertinoSwitch または Switch ウィジェットで endFlag をトグルできるフォームアイテムを追加する。
