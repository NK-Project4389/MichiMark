Designer Report — UI-24

ActionTime アクションボタン 大型化

4つのアクションボタンを横一列に並べた大型スクエアボタンへリデザイン。アクション名と直近の押下時刻を1ボタン内に収め、素早い操作と状況確認を両立する。

2026-04-17 対象: ActionTime ボトムシート 全トピック共通
01 Overview

課題と設計方針

現在の実装（ElevatedButton + Wrap）に対して、モバイル操作性・情報密度・ボトムシートUI全体のバランスを改善する。

現在の問題点: ElevatedButton + Wrap レイアウトはボタンサイズが小さく（テキスト幅依存）、「直近の押下時刻」が表示されない。アクションを押したのがいつか確認するにはログを見る必要がある。

提案の方針: 4ボタンを横一列に等幅で並べるグリッドレイアウトに変更。各ボタンに「アクション名（上部）＋ 直近押下時刻（下部・大きく）」を表示。ボトムシートのスペースを活かした縦幅確保で、タップ面積を大幅に拡大する。

Projection拡張: 各Actionに直近の押下時刻（lastLoggedAt）をProjectionに追加する必要がある。既存の ActionTimeLogProjection から逆引きするか、ActionDomain と logs を組み合わせて計算する。

07 Rationale

デザイン根拠

このリデザインを選択した理由。

ボタン幅を均等化

ElevatedButton + Wrapではボタン幅がテキスト長に依存し不均一。4等分グリッドにより「到着」「作業開始」「作業終了」「出発」が同じ幅になり、視覚的な安定感と一貫したタップ面積を実現する。

時刻表示で状況把握を即時化

現行は直近の押下時刻を確認するためにログセクションまでスクロールが必要。ボタン内に直接表示することで「いつ押したか」をボタンを見るだけで把握でき、特に現場作業中の素早い状況確認に有効。

ボトムシートの縦スペース活用

DraggableScrollableSheet（initialChildSize: 0.6）はかなりの縦スペースがある。88pxという高さはスペースを無駄にせず活用しながら、スクロール不要でアクションを完結させる設計。

既存Violetカラーパレットの継承

既存の_violetColor（#7C3AED）を踏襲し、ActionTimeのブランドカラーとして一貫性を保つ。既存の_ActionTimeButtonのVioletアイコンとも整合する。

08 Implementation Note

実装メモ

architect / flutter-dev への申し送り事項。

ActionTimeProjection への lastLoggedAt 追加

各Actionに「直近の押下時刻（lastLoggedAt: DateTime?）」を保持するための拡張が必要。現在の ActionTimeLogProjection（id, actionName, timestampLabel, transitionLabel）と ActionTimeDraft.logs の組み合わせから、アクションIDごとに最新のタイムスタンプを逆引きする仕組みを追加する。

Projection拡張案: ActionButtonProjection（新規）として、actionId, actionName, lastLoggedTimeLabel: String? を持つクラスを追加し、ActionTimeProjection に List<ActionButtonProjection> buttonItems を追加する。既存の availableActions とは別に持つ。

現行View変更箇所: ActionTimeView._ActionTimeContent 内の Wrap + ElevatedButton のマッピングを、新しいグリッドウィジェット（Row + Expanded）に置き換える。ロジック（ActionTimeLogRecorded のdispatch）は変わらない。

アクションボタン数が4以外の場合: TopicConfigによってアクション数が変わる可能性がある。ボタン数 ≤ 4 の場合は等幅グリッド、5以上の場合はWrapフォールバックにするか、architectが判断する。
