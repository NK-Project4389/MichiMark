COLOR PALETTE

採用カラー（2色固定）

今バージョンは白と #2B7A9B の厳密な2色制約。中間色・グラデーション・透過は使用しない。

コントラスト比: 白 on #2B7A9B = 約 5.6:1（WCAG AA 合格 / 4.5:1 以上）。 アイコン全体で2色のみを使用し、いかなる中間値・透過値も使わない。

設計制約（全案共通）

  • 3要素必須: ピン（地点マーク）・ミチ（道/曲線）・メジャー（目盛り/巻尺）が全案に含まれること
  • 2色のみ: 背景 #2B7A9B・前景 #FFFFFF のみ使用。中間色・グラデーション・光沢・影は禁止
  • Google マップ差別化: 孤立したドロップピン形状を主役にしない
  • 29pt 視認性: 最小サイズ（29 x 29pt）で3要素の形状が崩れないシンプルさ
  • App Store アイコン準拠: 1:1 正方形・丸角はOSが自動適用するため考慮不要

ICON PROPOSALS

3要素統合案 3パターン

各案は 160px / 60px / 29px の3サイズでプレビュー。29px は App Store 最小表示（29pt）相当の厳しい条件。

Plan A

メジャーテープ刺しピン

ピン: テープから立ち上がる ミチ: テープ本体が道路を表す メジャー: 目盛り刻み + 巻尺ケース形状

横長の巻尺テープを画面下部に水平に配置し、テープ自体が「道路（ミチ）」を象徴する。 テープの左端（スタート地点）にピンが垂直に刺さり、「この地点をマーク」という行為を表現する。 テープ上面には等間隔の目盛り刻みを入れ、メジャーであることを明示。 ピンはドロップピン形状ではなく「アップライトピン（頭が球の画鋲型）」にして Google マップとの差別化を図る。

Plan B

カーブ道路メジャー型

ピン: 道路の曲がり角に刺さる ミチ: S字カーブ路 メジャー: 道路脇の縦尺

左下から右上へ向かうカーブ道路（2本の白い平行線）が画面を斜めに横断し、「ドライブ・移動」を表現する。 道路の曲がり角にアップライトピンが立ち、「地点をマーク」を示す。 道路の左側に縦向きのメジャー尺（目盛り付き垂直帯）を配置し、「距離を測る」概念を補完する。 曲線によって Google マップのシンプルなピンアイコンとは全く異なるシルエットになる。

Plan C

メジャーボディ・ピン

ピン: 全体のシルエット ミチ: ピン内の道路ライン メジャー: ピンのボディが巻尺本体

円形の巻尺ケース（ロールメジャー）を「ピンの頭」として使い、ケースから伸びたテープが「ピンのスティック」になる構成。 テープ（スティック）面には目盛り刻みを入れ、メジャーであることを示す。 テープの先端（地面接触部）は細くなり、地点を示す先端点になる。 ピンの内部（ケース部）に道路の破線センターラインを小さく描き込み「ミチ」要素を補完する。 3要素が1つのオブジェクトに凝縮されるため最もシンプルなシルエットになる。

TYPOGRAPHY

アイコンのテキスト方針

アイコン本体にはテキストを入れない。スプラッシュ・App Store ページでブランド名を補完する。

App Store の最小表示（20x20px 程度）でテキストは完全に潰れるため、アイコンはシンボルのみで識別できる設計を維持する。 ブランド名「MichiMark」はアイコン下部のアプリ名ラベル（OS が表示）・スプラッシュ画面・App Store のスクリーンショットで補完する。

RATIONALE

デザイン根拠

なぜアップライトピン（画鋲型）を採用したか

Google マップのシンボルはドロップピン（水滴を逆にした形）。アップライトピンは「物理的に刺す」行為を直感的に表し、かつシルエットが球 + 棒という単純な形で小サイズでの視認性に優れる。MapKit・Apple マップも類似のドロップピンを使うため、差別化が明確になる。

メジャーテープを道路として兼用した理由（Plan A）

横長の帯形状は道路にもメジャーにも解釈できるデュアルシンボル。目盛り刻みを加えることでメジャー性が確定し、センターラインの破線を加えることで道路性が確定する。2要素を1オブジェクトで表現するため、アイコンの密度を低く保てる。

2色制約の意図

白 + 単色の2色制約はアイコンの輪郭を際立たせ、あらゆる壁紙・ウィジェット配色の上でも視認性を確保する。グラデーション・透過はダークモード・ライトモード・低輝度ディスプレイで予測不能な見え方になるリスクがある。2色固定はそのリスクをゼロにする。

コントラスト比の確認

  • 白 #FFFFFF on #2B7A9B: 約 5.6:1
  • WCAG AA 基準（4.5:1）合格
  • アイコン用途では規格基準より、視認の「強さ」を重視
  • 5.6:1 は 29pt の極小サイズでも十分な白/青コントラスト

v2 からの変更点（フィードバック反映）

v2 のピン単体では Google マップとの差別化が不十分という指摘を受け、全案で「ミチ・メジャー」の2要素を必須統合とした。カラーも v1 の多色パレットから「白 + #2B7A9B」の2色固定に変更。装飾（グラデーション・影・透過）を完全撤廃。

29pt 視認性の設計基準

App Store の最小アイコン表示（Spotlight 検索、設定画面）は 29pt。Plan A・C は大きな単純形状（球・帯・三角）で構成されるため 29pt でシルエットが崩れにくい。Plan B はカーブ2本線の間隔を確保する必要があり、Figma での実サイズ確認を特に推奨する。
