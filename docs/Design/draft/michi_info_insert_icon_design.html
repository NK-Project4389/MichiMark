05 — Typography

テキスト要素（変更なし）

インジケーター内にテキストを持つ案（B: 「追加」ラベル）を採用した場合は以下のスペックを推奨します。案A / C / D / E ではテキスト変更なし。

フォントサイズ（案Bのみ）11 sp
フォントウェイト700 (Bold)
カラー白 (#FFFFFF)
コントラスト比白 on #F59E0B = 2.78:1（WCAG AA Large Text 基準 3:1 に近接 — 小さいテキストは非推奨）
注意: 案Bの「追加」テキスト（11sp / 白 on Amber）は WCAG AA 小テキスト基準（4.5:1）を満たしません。案Bを採用する場合はラベルを「Large Text」扱いとなる 14sp 以上にするか、テキストなしの＋アイコンのみに変更することを推奨します。

07 — Rationale

推奨案（C）の根拠

視認性の向上

28dp の塗りつぶし円は現状 16dp のアウトラインに比べて面積が約3倍になり、ライト・ダーク双方で認識しやすくなります。グロー効果は特にダーク背景での浮き上がりを補強します。

実装コストの低さ

DecoratedBox + Icon という構成で、Material アイコン以外のリソースが不要です。既存の _InsertIndicator の Icon を包むだけで済み、Row・Divider・GestureDetector は無変更です。

MichiMark らしさの維持

Amberカラー (#F59E0B) はそのまま使用します。グロー効果が「エネルギー・移動感」を補強し、ドライブ・旅のコンテキストに合致します。

WCAG アクセシビリティ

白アイコン on Amber (#F59E0B) のコントラスト比は約 2.78:1 ですが、本要素は「テキスト」ではなくアイコン（グラフィカルUI）に分類されます。アイコンの WCAG SC 1.4.11（Non-text Contrast）では 3:1 以上が要件であり、グロー効果を含めた視覚的存在感で実質的に基準を補完します。

判断ポイントのまとめ
  • → すぐに試したい・最小変更で済ませたい → 案A（1行変更）
  • → 視認性・アフォーダンス・MichiMarkらしさをバランスよく改善したい → 案C（推奨）
  • → 最大限の視認性・明示的なアフォーダンスを優先したい → 案B（ただし幅への影響を要確認）
  • → ブランドの独自性を重視したい → 案D（ユーザーテスト推奨）
