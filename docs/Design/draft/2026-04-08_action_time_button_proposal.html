Section 05

デザイン根拠・推奨案

DESIGNER RECOMMENDATION

推奨: 案B — アイコンボタン + ボトムシート

MichiInfo タイムラインはスクロールが主体のリスト画面。各カードに複数のアクションボタンを 直置きすると縦スペースを圧迫し、タイムラインの「流れを読む」体験が損なわれる。 案B はカード右上の ⚡ アイコン（直径 44pt）をタップするだけでボトムシートが開くため、 1回の意図的タップ → 全アクション確認 → 選択 という安全な操作フローを保てる。

既存の ActionTimeView（現在状態カード・アクション Wrap・休憩トグル）をボトムシートとして ラップするだけで流用できるため、実装コストが最も低く、表示内容の一貫性も保たれる。 ドライブアプリ・地図アプリ（Apple Maps・Google Maps）でもアクションは「タップ → シート展開」 が標準パターンとなっており、ユーザーへの学習コストも低い。

タイムライン閲覧
→
Mark カードの ⚡ をタップ
→
ボトムシートが開く
→
現在状態を確認
→
アクションを選択
→
シートが閉じてタイムラインに戻る
案B の追加検討ポイント
アイコンの視認性
⚡ アイコンボタンはカード右上に配置し、Mark Primary（Teal）の上に Violet の円形ボタンを重ねることで既存ラベルと明確に区別する。
状態バッジのオンカード表示（補助）
現在の ActionTime 状態（「移動中」など）を小さなバッジとして Mark カードに 常時表示すると、シートを開かずとも状態把握が可能になる。案B との組み合わせを推奨。
ボトムシートの高さ
half-height（画面の約 50%）が適切。全アクションが収まり、タイムラインの コンテキストも半分見えた状態を保てる。
アクセシビリティ
⚡ ボタンの Tooltip / セマンティクスラベルに「ActionTime を記録」を付与し、 スクリーンリーダー対応を確保する。

デザイン要件の叩き: ActionTime アクションボタン UI

作成日: 2026-04-08 / 作成者: designer / 参照レポート: docs/Design/draft/2026-04-08_action_time_button_proposal.html
概要

MichiInfo タイムライン画面の Mark カードに ActionTime 記録のトリガーとなる UI を追加する。 推奨案は 案B（アイコンボタン + ボトムシート）。 既存の ActionTimeView をボトムシートとして流用することで実装コストを抑えつつ、 タイムラインの情報密度を保つ。

採用案の選択（確認事項）
  • 案B（アイコンボタン + ボトムシート）を採用する ← designer 推奨
  • または 案A（カードにボタン直置き）を採用する
  • または 案C（スワイプ）を採用する
  • または 新たな案を検討する
カラー定義（案B 採用前提）

以下のカラーを ActionTime 専用トークンとして追加する。

actionPrimary: #7C3AED actionDark: #5B21B6 actionLight: #A78BFA actionTint: #EDE9FE actionTintLight: #F5F3FF
stateMoving: #0EA5E9 stateStaying: #10B981 stateWorking: #F59E0B stateResting: #6B7280
変更項目
  • Mark カードのヘッダー右端に ActionTime アイコンボタン（⚡ Violet 円形、直径 44pt）を追加
  • Mark カードに現在 ActionTime 状態バッジを小さく常時表示（オプション: 確認必要）
  • ⚡ タップで既存 ActionTimeView をボトムシートとして表示
  • ボトムシートは half-height（画面約 50%）で展開
  • ボトムシート上部にアクション対象 Mark 名を表示
  • アクション実行後にボトムシートを自動クローズしタイムラインへ戻る
  • カラートークン（上記 9色）を追加定義
