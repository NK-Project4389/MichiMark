挿入ボタン カラー定義

既存のMark（Teal）・Link（Emerald）カラーに対して、挿入アクションはAmber（#F59E0B）を採用する。Amberは「アクション誘発色」として広く認知されており、TealとEmeraldの寒色系に対してコントラストを持ちながらも、ブランドイメージである「ドライブ・旅」のエネルギー感と調和する。

Amberを採用する理由: Mark（Teal）でもLink（Emerald）でもない第3色として「追加・挿入」アクションを色で区別する。白背景上でのコントラスト比 — Amber (#F59E0B) on White は低い（2.5:1）ため、ボタン内テキストには白ではなく Dark (#7A4A00 近辺) を使用するか、Amber背景＋白文字は大きいアイコン（＋記号）のみに限定する。WCAG AAを満たすには塗りつぶしボタン＋白アイコンの組み合わせを基本とする。

メリット

推奨シーン

Insert Type Selection

挿入種別選択UI — ボトムシート

どのパターンでも「＋」タップ後に「MarkかLinkか」を選択するUIが必要。ボトムシート形式を推奨。シンプルな2択でユーザーに迷いを与えない。

設計メモ: ボトムシートでの2択UIを採用することで、挿入ポイント（どこに）と挿入種別（何を）の選択を2ステップに分離できる。これにより各ステップのUIをシンプルに保てる。「Mark後 → Linkのみ選択可」など制約をつける場合はグレーアウトで表現する。

Designer Recommendation

デザイナー総合推奨案

推奨: Pattern 3 × ボトムシート選択

FAB型 + 挿入モード + ボトムシート種別選択

MichiMarkのタイムラインは「閲覧」が主目的であり、「挿入」は編集操作として明確に区別されるべきアクションである。 Pattern 3（FAB型）は通常閲覧時にタイムラインを一切汚さず、FABタップによって初めて「挿入モード」に入るという明確なUXフローを提供できる。 これはiOS・Androidいずれのモバイル慣習とも整合しており、ユーザーへの学習コストが低い。

挿入ポイントのインジケーター（小さなAmberドット＋点線）は、挿入モード中にのみタイムライン上に出現させることで、 C-2スタイルのクリーンなビジュアルを損なわない。ポイントタップ後にボトムシートで「Mark / Link」の2択を提示する構成は、 操作の意図を明確化しつつ最小のUIで完結できる。

短期フォールバック案: Pattern 1（常時表示型）は実装コストが最も低く、初期リリースやプロトタイプ段階での採用に向く。 カード数が少ないMichiMarkの初期ユースケースでは視覚的混雑も許容範囲内であり、まずPattern 1で機能実装し、 UXが安定したらPattern 3に段階移行するアプローチも有効である。

カラーについての補足:
挿入ボタンのAmber（#F59E0B）採用を推奨する。理由は以下の3点。
1. Mark（Teal）・Link（Emerald）との明確な色区別ができ、「追加アクション」として独立した役割を持てる。
2. 寒色系（TealとEmerald）に対して暖色系Amberは視覚的に浮かび上がり、ボタンの誘目性が高い。
3. 代替案の「白抜き＋Teal枠」はMarkドットと色が被り、挿入ボタンとMarkが混同される恐れがあるため不推奨。
