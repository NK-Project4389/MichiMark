## 2. 改善の背景・目的

App Storeでユーザーが見るのは「プロモーションテキスト → スクリーンショット1枚目 → 説明文冒頭3行」の順。
現状の説明文は機能説明から始まっているため、「自分のための問題を解決してくれるアプリ」という印象を持たせるまでに時間がかかる。
スクリーンショットも素の画面キャプチャのため、コンテキストなしで見た見込みユーザーには何をするアプリか伝わりにくい。

改善の方向性は「誰のための・何ができる・どう変わる」を3秒で伝える構成にシフトすること。

## 3. 改善草案

### 3-1. プロモーションテキスト（改善案）

**現状（85文字）**
```
移動のすべてを記録する。地点・区間・費用・燃費をひとつのアプリで管理。ドライブ・ツーリング・旅行・仕事の移動に。タイムライン形式で「あの日どこ行って何したか」が一目でわかる。
```

**改善案A：Before→After型（82文字）**
```
「あの日どこ行って、いくらかかったっけ？」をなくすアプリ。地点・費用・燃費をタイムラインで記録。ドライブ・ツーリング・旅行・仕事の移動に。
```

**改善案B：問いかけ型（79文字）**
```
ドライブの記録、ちゃんと残ってる？地点・走行距離・旅費をひとつのアプリで完結。タイムライン形式で移動の流れを丸ごと振り返れる無料アプリ。
```

**推奨**: 改善案A。「あの日どこ行って何したか」という感情的なフックを残しつつ、Beforeを明示して共感を引き出す。

### 3-2. 説明文（改善草案）

> 冒頭3行が最重要。「もっと見る」タップ前に表示される部分を優先改善。

**現状の冒頭**
```
移動の記録・集計・管理をワンアプリで完結。

MichiMarkは、ドライブ・ツーリング・旅行・仕事の移動を記録して集計できるログアプリです。
```

**改善案（冒頭3行）**
```
あの日のドライブ、どこ行って何キロ走って、いくらかかったか覚えてる？

MichiMarkはその「記録」をかんたんにするアプリです。地点・走行距離・旅費・燃費をタイムライン形式で記録して、後からいつでも振り返れます。

ツーリング・ドライブ・旅行・仕事の移動まで、あらゆる移動をひとつにまとめて管理。
```

**説明文（全体・改善版）**
```
あの日のドライブ、どこ行って何キロ走って、いくらかかったか覚えてる？

MichiMarkはその「記録」をかんたんにするアプリです。地点・走行距離・旅費・燃費をタイムライン形式で記録して、後からいつでも振り返れます。

ツーリング・ドライブ・旅行・仕事の移動まで、あらゆる移動をひとつにまとめて管理。

【こんな悩みを解決します】

・ツーリングのルートや立ち寄りスポットを記録しておきたい
・旅行の費用をメンバーと割り勘するのが毎回面倒
・仕事の移動距離・交通費を月末にまとめて計算するのがつらい
・給油ごとの燃費を自分で計算している
・ドライブの思い出が薄れていく気がする

【主な機能】

■ タイムライン形式のルート記録（MichiInfo）
地点（マーク）と区間（リンク）を登録してルートを記録。「自宅出発 → SA → 目的地」の流れが一画面で見えます。距離・時間も区間ごとに記録可能。

■ 旅費・精算の自動計算
高速代・キャンプ場代・食費などをメンバーごとに登録すると、誰がいくら払ってどのくらい受け取るかを自動で計算。旅行後の精算がスムーズになります。

■ 燃費・移動コストの集計
累積メーターと給油量を入力するだけで燃費と移動コストを自動集計。ガソリン代がひと目でわかります。

■ トピック別の整理
「ツーリング」「旅費可視化」「仕事移動」などのトピックでイベントを分類。用途に合わせた情報だけが表示されます。

■ タグ・メンバーの候補表示
よく使うメンバーやタグをあらかじめ登録すれば、入力時にすぐ候補が表示。入力の手間を最小限に抑えます。

■ カスタム数値キーパッド
距離・金額の入力に特化したキーパッドを採用。四則演算もその場でできます。

【プライバシー】

MichiMarkはすべてのデータをお使いのデバイス内にのみ保存します。個人情報の外部送信・収集は行いません。

プライバシーポリシー: https://michimark-web.vercel.app/privacy.html
サポート: https://michimark-web.vercel.app/support.html
```

### 3-3. キーワード（改善案）

**現状（48文字）**
```
ドライブ,ツーリング,旅行,移動記録,ログ,燃費,旅費,走行距離,精算,移動コスト,タグ,訪問
```

**改善案（96文字・最大活用版）**
```
ドライブ,ツーリング,旅行,移動記録,燃費,旅費,走行距離,精算,移動コスト,訪問,バイク,マイカー,経費,交通費,給油,キャンプ
```

改善の根拠：
- 「バイク」「マイカー」を追加：ツーリング層とドライブ層の双方をカバー
- 「経費」「交通費」を追加：仕事の移動記録用途の検索流入を狙う
- 「給油」を追加：燃費管理を探すユーザーへのアプローチ
- 「キャンプ」を追加：旅行×アウトドア層への訴求
- 「ログ」を削除：「移動記録」で意味を包含、かつ重複性低い
- 「タグ」を削除：機能名であり検索されにくい
- カテゴリ名（ナビゲーション・旅行）・アプリ名との重複は除外済み

### 3-4. スクリーンショット改善方針

#### 現状の評価

| 枚 | 内容 | 課題 |
|---|---|---|
| 1 | イベント一覧（生画面） | テキストオーバーレイなし・アプリ名すら見えない状態で何のアプリか伝わらない |
| 2 | タイムライン画面 | 最も差別化できる画面だが1枚目ではない。ファーストビューに持ってくることも検討価値あり |
| 3 | 集計・精算（概要タブ） | 数値が並んでいて説得力がある。良好 |
| 4 | 支払いタブ（支払い一覧） | 精算結果（誰がいくら払うか）まで見えると訴求力が上がる |
| 5 | 地点詳細・給油入力 | 「給油記録」というニッチ機能にフォーカスしており、初見ユーザーへの訴求力が低い。汎用的な地点入力画面への変更を検討 |

#### 改善方針（designer連携推奨）

**1枚目**：テキストオーバーレイを追加する。キャッチコピー「移動の記録を、もっとかんたんに」を画面上部に白文字で重畳。

**2枚目**：現状維持。キャッチコピー「地点・区間・時間が一画面に」のオーバーレイを追加。

**3枚目**：現状維持。キャッチコピー「走行距離・移動コストを自動集計」のオーバーレイを追加。精算結果も見えるスクロール位置に調整。

**4枚目**：支払い一覧→精算結果（収支バランス・支払いごとの精算）の画面に差し替えを検討。「誰がいくら払うか自動計算」の訴求力が高い。

**5枚目**：地点詳細の給油画面ではなく、タグ・メンバーのインライン選択UIが見える画面（入力しやすさの訴求）に差し替えを推奨。
