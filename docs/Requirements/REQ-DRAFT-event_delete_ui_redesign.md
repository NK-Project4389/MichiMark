# 要件書（下書き）: イベント削除UI変更

- **要件ID**: REQ-DRAFT-event_delete_ui_redesign
- **作成日**: 2026-04-12
- **担当**: product-manager
- **ステータス**: 下書き（確定前）
- **種別**: UI改善（UI-1）

---

## 1. 背景・目的

現在、イベント一覧画面でカードを左スワイプすると削除ボタンが表示される操作になっている。
この操作を廃止し、イベント詳細画面のヘッダに削除アイコンを移動する。

誤削除防止と操作の一貫性向上が目的。

---

## 2. 変更仕様

### 2.1 廃止する操作

- イベント一覧画面（EventListView等）のカードスワイプによる削除表示・削除操作

### 2.2 追加する操作

- イベント詳細画面のヘッダ右側に削除アイコンを追加
- アイコンの直下または近傍に「イベント削除」のテキストラベルを追記
  - 他画面の削除アクション（Mark削除、Link削除など）と異なることを明示するため

### 2.3 削除フロー（変更後）

イベント詳細画面 → ヘッダ右側の削除アイコンをタップ → （確認ダイアログの要否はSpec作成時に確認）→ イベント削除

---

## 3. 影響範囲

| 画面 | 変更内容 |
|---|---|
| イベント一覧画面 | `flutter_slidable` によるスワイプ削除UI撤去 |
| イベント詳細画面 | ヘッダ右側に削除アイコン＋「イベント削除」ラベル追加 |

---

## 4. 未確定事項（Spec作成時に確認）

- 削除前の確認ダイアログの要否
- 削除アイコンのデザイン（ゴミ箱アイコン等）

---

## 5. 対象外

- イベント削除時のカスケード削除仕様（既存実装に準ずる）
- 削除取り消し（Undo）

---

## 6. テストシナリオ概要（UI-1）

1. イベント一覧でカードをスワイプしても削除ボタンが表示されないことを確認
2. イベント詳細画面のヘッダ右側に削除アイコンが表示されることを確認
3. 削除アイコン付近に「イベント削除」のラベルが表示されることを確認
4. 削除アイコンをタップするとイベントが削除されることを確認
