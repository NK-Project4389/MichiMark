# 要件書: PaymentInfo UI 改善 + 精算セクション追加

## 概要
PaymentInfo タブのリスト表示を視認性向上のため改善し、概要タブに伝票単位の精算セクションを追加する。

## スコープ
- PaymentListTile のデザイン刷新
- 概要タブ（TravelExpenseOverviewView 等）への「支払いごとの精算」セクション追加

---

## 1. PaymentListTile 改善（案A ベース・修正版）

### 現状の問題
- 支払者・割り勘メンバー・メモが subtitle の1行に羅列されており視認性が悪い
- 割り勘メンバーが多い場合に名前が途切れる

### 新しいレイアウト（3〜4行構成）

```
[💳アイコン]  ¥3,500                       （行1: 金額 Bold）
              支払  [太郎]                  （行2: "支払" プレーンテキスト + 名前チップ Teal）
              割り勘  [花子] [健太]          （行3: "割り勘" プレーンテキスト + 名前チップ Emerald・Wrap折り返し）
              焼肉ランチ                     （行4: メモ・任意表示）
```

### デザイン仕様

| 要素 | スタイル |
|---|---|
| 行1: 金額 | `titleMedium` または `bodyLarge`・Bold |
| 行2: "支払" ラベル | プレーンテキスト（背景色なし）・`bodySmall` |
| 行2: 支払者名チップ | Teal 背景（`#2B7A9B` 系）・白テキスト・角丸 |
| 行3: "割り勘" ラベル | プレーンテキスト（背景色なし）・`bodySmall` |
| 行3: 割り勘メンバーチップ | Emerald 背景（`#2E9E6B` 系）・白テキスト・角丸・Wrap折り返し |
| 行4: メモ | `bodySmall`・`text-secondary`・italic（メモが空の場合は非表示） |
| 割り勘なし | 行3 非表示 |

---

## 2. 「支払いごとの精算」セクション追加（概要タブ）

### 配置
TravelExpenseOverviewView の「収支バランス」セクション直下に追加。

### 表示内容
各伝票（PaymentItemProjection）ごとに、参加者間の精算金額を表示する。

```
支払いごとの精算

┌──────────────────────────────────┐
│ 焼肉ランチ                ¥3,500 │
│  花子 → 太郎 : ¥1,750            │
│  健太 → 太郎 : ¥1,750            │
└──────────────────────────────────┘
┌──────────────────────────────────┐
│ （メモなし）               ¥800  │
│  花子 → 太郎 : ¥400             │
└──────────────────────────────────┘
```

### 精算ロジック
- 支払者が全額立て替え、割り勘メンバーが均等割りして支払者に返す
- 割り勘メンバーがいない伝票は精算セクションを非表示
- メモが空の場合はタイトルを「支払 #N」（N は伝票番号）で代替表示

### デザイン仕様

| 要素 | スタイル |
|---|---|
| セクションタイトル | 「支払いごとの精算」・`titleSmall` |
| 伝票ヘッダー | タイトル（左）+ 金額（右）・`bodyMedium` |
| 伝票金額背景 | Teal tint 背景 |
| 精算行 | `支払う人（赤色）→ 受け取る人（緑色）: 金額` |
| カード間余白 | 8px |
