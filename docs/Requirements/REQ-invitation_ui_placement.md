# 要件書：イベント招待機能 UI配置（INV-UI-1）

## 概要

イベント詳細画面に招待機能への導線が存在しないため、概要タブまたはヘッダ上に招待機能へのアクセスポイントを配置する。
イベントオーナーは招待リンクの生成・共有（INV-4）、それ以外のユーザーは招待コード入力（INV-3）に素早くアクセスできるようにする。

## ユーザーストーリー

- イベントオーナーとして、イベント詳細画面から直接メンバーを招待したい（毎回EventListPageに戻る手間なく）
- イベントオーナーとして、「メンバーを招待」の操作起点がイベントの文脈内にあることで、招待対象イベントを間違えずに招待したい
- 参加検討中のユーザーとして、招待コードを受け取ったときにイベント詳細画面から直接コード入力画面へ進みたい

## 前提

- INV-1（招待バックエンドAPI）の完了が必須
- INV-2（招待中間Webページ）の完了が必須
- INV-3（招待コード入力画面・Flutter）の完了が必須
- INV-4（招待リンク生成・共有）の実装と同時またはそれ以前に着手する

## 権限モデル

| 権限 | 招待リンク生成・共有ボタン（INV-4への導線） | 招待コード入力ボタン（INV-3への導線） |
|---|---|---|
| `owner` | 表示する | 非表示（自分のイベントのため不要） |
| `editor` | 非表示 | 表示する |
| `viewer` | 非表示 | 表示する |
| 未参加（権限なし） | 非表示 | 表示する |

## スコープ

### 対象

#### 概要タブ内への配置（推奨案）

- 概要タブ内の基本情報セクション下部に招待ボタンを追加する
- owner 向け:「メンバーを招待」ボタン → INV-4（招待設定シート）を開く
- 非owner 向け:「招待コードを入力」ボタン → INV-3（招待コード入力画面）へ遷移

#### AppBar（ヘッダ）への配置（代替案）

- EventDetail の AppBar に招待アイコンボタンを追加する
- 上記いずれかをarchitectが設計判断する（両案を併用しない）

### 対象外

- EventListPage の AppBar にある既存の招待コード入力ボタンの削除・変更
  - 既存動線はそのまま残す（削除は別タスクで判断する）
- 招待リンク生成・共有ロジック本体（INV-4のスコープ）
- 招待コード入力ロジック本体（INV-3のスコープ）
- 招待メンバー一覧表示

## 受け入れ条件

1. イベント詳細画面（概要タブまたはAppBar）に招待機能への導線ボタンが存在する
2. owner でログイン中は「メンバーを招待」ボタンのみ表示され、タップすると INV-4 の招待設定シートが開く
3. editor / viewer / 未参加ユーザーでログイン中は「招待コードを入力」ボタンのみ表示され、タップすると INV-3 の招待コード入力画面へ遷移する
4. owner に「招待コードを入力」ボタンは表示されない
5. 非owner に「メンバーを招待」ボタンは表示されない
6. ボタンには適切な Widget Key が付与されており Integration Test から参照可能である
