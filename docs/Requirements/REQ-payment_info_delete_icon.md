# 要件書: PaymentInfo カード削除UI変更

- **要件ID**: REQ-payment_info_delete_icon
- **作成日**: 2026-04-12
- **担当**: product-manager
- **ステータス**: 確定
- **種別**: UI改善（UI-4）

---

## 1. 背景・目的

PaymentInfo画面のカード削除操作を、スワイプ方式から常時表示アイコン方式に変更する。

MichiInfo（UI-3）と操作方式を統一し、UIの一貫性を確保する。
スワイプへの気づきにくさを解消し、削除操作を直感的にする。

---

## 2. 要件一覧

| 項目 | 内容 |
|---|---|
| 廃止する操作 | `flutter_slidable` によるスワイプ削除ボタン表示 |
| 追加する操作 | PaymentInfoカード右側に削除アイコンを常時表示 |
| 確認ダイアログ | **なし**（タップ即削除 ← 既存仕様を維持） |
| 削除ロジック | 変更なし（REQ-payment_info_card_delete.md に準ずる） |
| アイコンデザイン | UI-3（MichiInfo削除アイコン）と統一するデザインを採用 |

---

## 3. 影響範囲

| 画面 | 変更内容 |
|---|---|
| PaymentInfo（PaymentCard） | スワイプUI撤去・カード右側に削除アイコン常時表示 |

---

## 4. 対象外

- 削除後のデータ処理（既存実装に準ずる：REQ-payment_info_card_delete.md 参照）
- 削除取り消し（Undo）
- 確認ダイアログの追加

---

## 5. テストシナリオ

| TC-ID | シナリオ | 期待結果 |
|---|---|---|
| TC-PID2-001 | PaymentInfoカードを左スワイプする | 削除ボタンが表示されない |
| TC-PID2-002 | PaymentInfoカードの右側を確認する | 削除アイコンが表示されている |
| TC-PID2-003 | 削除アイコンをタップする | 該当カードが即座に削除されダイアログは表示されない |
