# 要件書: タグレコメンド常時表示・最近使った順ソート

**要件ID**: REQ-tag_recommend_display
**作成日**: 2026-04-08
**ステータス**: 未実装
**関連要件**: REQ-tag_inline_suggest（タグインラインサジェスト入力・実装済み）

---

## 背景・目的

現状のタグインライン入力では、テキストフィールドに入力したときのみサジェストが表示される（FR-003: 入力が空の場合はサジェスト非表示）。
マスタに登録済みのタグを「レコメンド」として常時表示することで、タグ選択のUXをさらに向上させる。
また、最近使ったタグを先頭に表示することで、よく使うタグを素早く選択できるようにする。

---

## ユーザーストーリー

- As a ユーザー, I want to タグ入力フィールドを開いたとき（入力なしでも）マスタタグをレコメンドとして表示してほしい, so that 何を入力すればいいかわからないときでも素早くタグを選べる
- As a ユーザー, I want to 最近使ったタグが先頭に表示されてほしい, so that よく使うタグを毎回探す手間が省ける
- As a ユーザー, I want to タグを入力して候補が絞り込まれてほしい, so that 特定のタグを素早く見つけられる

---

## 機能要件

### FR-TAG-001: タグ入力セクションのUI構成（3ブロック構成）

編集モードのタグセクションを以下の3ブロックで構成する：

```
[ 登録済みのタグ ]        ← 既存のチップ表示（変更なし）
[ レコメンド ]           ← NEW: マスタ全件を常時表示（絞り込み対応）
[ タグを入力して追加... ] ← 既存のテキスト入力フィールド（変更なし）
```

### FR-TAG-002: レコメンドの常時表示

- テキスト入力フィールドの上（または下）に「レコメンド」セクションを表示する
- 入力フィールドが空のとき: マスタの全タグ（未選択のもの）を表示する
- 入力フィールドに文字があるとき: 入力文字で部分一致フィルタをかけた結果を表示する
- レコメンドセクションには「レコメンド」ラベルを表示する
- 表示対象: 既に選択済みのタグは除外する

### FR-TAG-003: 最近使った順でのソート

- レコメンドの表示順は `TagDomain.updatedAt` の降順（新しいもの順）とする
- タグをレコメンドから選択したとき、そのタグの `updatedAt` をマスタ側でも更新する（使用日時として記録）
- 手動入力で新規マスタ登録したタグは `createdAt = updatedAt = now` で作成されるため自動的に先頭になる

### FR-TAG-004: レコメンドからの選択

- レコメンドリストのアイテムをタップするとタグが選択状態になる（既存の `BasicInfoTagSuggestionSelected` を流用）
- 選択後、入力フィールドはクリア（既存動作と同じ）
- 選択後、レコメンドは「全件から選択済みタグを除いた状態」に更新する

---

## 非機能要件

- マスタタグは `BasicInfoBloc` 内に `allTags` としてキャッシュ済み（DB再取得なし）
- ソートはメモリ内で実施（DB側ソートは不要）
- レコメンドが多い場合は `maxHeight` を設けスクロール可能にする（最大4件分の高さで切り捨て等はしない）

## 変更箇所

| ファイル | 変更種別 | 内容 |
|---|---|---|
| `basic_info_bloc.dart` | 変更 | `_onTagInputChanged`: 空入力時も全マスタタグをサジェストとして返す（updatedAt降順）|
| `basic_info_bloc.dart` | 変更 | `_onTagSuggestionSelected`: 選択タグの `updatedAt` を更新してマスタ保存 |
| `basic_info_bloc.dart` | 変更 | `_onEditModeEntered`: 初期サジェストとして全マスタタグ（updatedAt降順）を設定 |
| `basic_info_view.dart` | 変更 | `_TagInputSection`: レコメンドセクションに「レコメンド」ラベルを追加 |

---

## 対応しないもの

- タグマスタ画面（設定 → タグ）側のUI変更
- マーク詳細・リンク詳細等の他画面でのタグ入力変更
- レコメンドの件数上限（全件表示）

---

## 備考

- 既存の `_onTagInputChanged` で `input.isEmpty` のときに `tagSuggestions: []` をemitしていた箇所を変更する
- `tagSuggestions` フィールドは「レコメンド兼フィルタ済みサジェスト」として兼用する
