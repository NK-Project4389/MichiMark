# 要件書: MichiInfo レイアウト変更

## 概要

MichiInfo Feature の一覧表示を、現行の `ListTile` ベースのリストから「道」を視覚的に表現するタイムライン型レイアウトへ変更する。
縦罫線と吹き出し型カードにより、ドライブルートの地点（Mark）・区間（Link）の構造を直感的に伝えることを目的とする。

---

## ユーザーストーリー

- ユーザーとして、地点（Mark）が縦のラインに沿ったドット（●）で視覚的に示されることで、ドライブルートの流れを一目で把握したい
- ユーザーとして、地点カードが吹き出し型で表示されることで、ドットとカードの対応を直感的に理解したい
- ユーザーとして、区間（Link）が登録されている範囲の縦線が太くなることで、区間の始まりと終わりを視覚的に認識したい
- ユーザーとして、地点間のメーター差分と区間距離が並列で表示されることで、各区間の距離情報を比較しながら確認したい
- ユーザーとして、距離凡例が右上に常時表示されることで、距離表示の意味（メーター差分 vs 区間距離）をいつでも確認できたい

---

## 機能要件

### FR-01: 縦ライン・ドット表現

- 画面左側に1本の縦罫線を配置し「道」を表現する
- Mark（地点）の位置に対応するドット（●）を縦線上に重ねて配置する
- 縦線・ドットのカラーは `colorScheme.onSurface` 系を使用する
- ドットのサイズは縦線より明確に大きく視認しやすいこと
- Link（区間）が登録されている場合、その区間に対応する縦線の部分のみ **太線** で表示する（色変更なし、線幅のみ変更）
- 太線区間の判定は Link の `startMarkId` / `endMarkId`（または順序インデックス）に基づく

### FR-02: 地点カード（吹き出し型）

- Mark に対応するカードは **三角ポインター付き吹き出し形** とする
- 三角ポインターはカードの左側に配置し、縦線上のドットに向かって伸びる
- カード内に表示する情報:
  - Mark 名称（`title`）
  - メーター値（`meterValue`、単位: km）
  - Mark に紐づく属性（給油・支払など、現行 ListTile と同等の情報）

### FR-03: 区間カード + 太線表現

- Link に対応するカードは通常の角丸カード（吹き出しポインターなし）とする
- カード内に表示する情報:
  - Link 名称（`title`）
  - 区間距離（`distanceValue`、単位: km）
  - Link に紐づく属性（給油・支払など、現行 ListTile と同等の情報）
- FR-01 の太線表現と合わせて表示する

### FR-04: 距離表示（右側）

- 地点と地点の間の右側に距離情報を縦に表示する
- 距離情報の構成:
  - **メーター差分**（前後の Mark の `meterValue` の差分）: プライマリ表示（大・太字・`onSurface` 色）
  - **区間距離**（Link の `distanceValue`）: セカンダリ表示（小・グレー）
- 地点間に Link が存在する場合は両方を縦に並べて表示し、存在しない場合はメーター差分のみ表示する
- 矢印（↕）またはそれに準じる視覚要素で距離範囲を表現する
- メーター差分の計算: `meterValue[n+1] - meterValue[n]`（差分がマイナスの場合も表示する）

### FR-05: 距離凡例（右上）

- 画面右上に距離表示の凡例を常時表示する
- 凡例の内容:
  - 黒（`onSurface`）: メーター差分
  - グレー: 区間距離
- 凡例はテキストラベル + 色見本の小さなインジケーターで構成する
- 凡例はリストのスクロールに関わらず固定表示（画面スタック上部）とする

---

## 非機能要件

- カラーは Flutter デフォルト `colorScheme` の範囲内で使用し、カスタムカラーを定義しない
- ライトテーマのみ対応（ダークテーマ対応はスコープ外）
- 縦線・ドット・太線は Canvas または `CustomPaint` で描画することを想定するが、実装詳細は Spec で定義する
- 既存の MichiInfo Bloc / Event / State の構造は変更しない（Widget 層のみ変更）
  - ただし、メーター差分の計算に必要な Projection の変更は Spec で判断する

---

## スコープ外

- ダークテーマ対応
- 縦線・ドット・カードのアニメーション
- Mark / Link の並び替え操作
- メーター差分のマイナス値に対する警告表示
- 距離単位の切り替え（km 固定）

---

## 受け入れ条件

- [ ] 縦線が画面左側に表示され、各 Mark の位置にドットが重なって表示される
- [ ] Link が存在する区間の縦線が他の区間より太く表示される
- [ ] Mark カードが三角ポインター付き吹き出し形で表示される
- [ ] Link カードが角丸の通常カードで表示される
- [ ] 地点間の右側にメーター差分（大・太字）が表示される
- [ ] Link が存在する区間では区間距離（小・グレー）がメーター差分の下に表示される
- [ ] 画面右上に凡例（黒 = メーター差分、グレー = 区間距離）が常時表示される
- [ ] 既存の Mark / Link タップ操作（詳細画面への遷移）が引き続き動作する

---

## 備考

- 現行実装: `ListTile` + `Icons.place`（Mark）/ `Icons.route`（Link）
- メーター差分の計算ロジックは Projection 層または Bloc で担当するか、Spec 作成時に設計判断を行う
- SwiftUI 版には本レイアウトの実装はない（Flutter 版の新規デザイン）
