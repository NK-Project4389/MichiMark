# 要件書: UI-19 訪問作業 Mark カード UI 改善

作成日: 2026-04-18
作成者: product-manager
バージョン: 1.0
参照デザイン叩き（最新）: `docs/Design/draft/visit_work_action_button_center_design.html`
参照デザイン叩き（旧・課題分析）: `docs/Design/draft/visit_work_action_badge_redesign.html`

---

## 概要

MichiInfo 訪問作業（visitWork）トピックの Mark カード UI を改善する。
現状のカードは、アクションボタン（28x28dp）と削除ボタン（36x36dp）がカード右端に密集しており、誤タップリスクが高い。
中央配置レイアウト（左: 日付+バッジ / 中央: 地点名+アクションボタン / 右: 削除ボタン）を採用し、操作性と視認性を向上させる。
合わせて、訪問作業の「休憩」アクションの削除およびアクション表示順の変更も行う。

---

## ユーザーストーリー

- ユーザーとして、作業現場でアクションボタンを確実にタップしたい
  -> アクションボタンが中央に大きく配置され、削除ボタンと物理的に離れるため誤タップが減る
- ユーザーとして、削除ボタンを誤って押したくない
  -> 削除ボタンが小さく（28x28dp）・薄く（opacity 0.6）・目立たない色（グレー系）で表示される
- ユーザーとして、訪問作業の状態（待機中・作業中など）をすぐに確認したい
  -> 状態バッジが左カラムに見やすく配置される（11px / semibold）

---

## 機能要件

### REQ-VWA-01: カードレイアウト変更（中央配置）

visitWork トピック（`topicConfig.showActionTimeButton == true`）の Mark カードのみ対象。
カード内を以下の Row 構成に変更する。

| 領域 | 内容 | サイズ | 配置 |
|---|---|---|---|
| タイムラインドット | 紫色の丸 | 10dp | 固定幅 |
| 左カラム | 日付テキスト（上）+ 状態バッジ（下） | 54dp 固定幅 | Column, alignStart |
| 中央カラム（Expanded） | 地点名（上）+ アクションボタン（下） | 残り幅すべて | Column, alignCenter |
| 右カラム | 削除ボタン（小さめ・薄め・グレー系） | 28dp 固定幅 | 固定幅, alignEnd |

- movingCost / travelExpense トピックのカードレイアウトは変更しない
- Link（区間）カードは変更しない

### REQ-VWA-02: アクションボタン仕様（中央配置）

| 項目 | Before | After |
|---|---|---|
| サイズ | 28 x 28dp | 横 padding:16dp + テキスト幅 x 高さ 36dp |
| タップ領域 | 28dp（WCAG非準拠） | 44dp以上（WCAG準拠） |
| ラベル | アイコンのみ | アイコン + テキスト「アクション」 |
| 角丸 | 8dp | 10dp |
| 背景色 | #7C3AED | #7C3AED（変更なし） |
| 文字色 | 白 | 白（変更なし） |
| フォントサイズ | - | 13px / bold |

### REQ-VWA-03: 削除ボタン仕様（案C採用: カード内常時表示・目立たない配置）

削除ボタンの配置は案C（カード内に常時表示）を採用する。ただし、現状の赤色が目立ちすぎるため色味をグレー系に変更する。

| 項目 | Before | After |
|---|---|---|
| サイズ | 36 x 36dp | 28 x 28dp（縮小） |
| 不透明度 | 1.0 | 0.6 |
| アイコンサイズ | 20dp | 14dp |
| 背景色 | #FEE2E2（薄赤） | #F3F4F6（Gray 100） |
| アイコン色 | #DC2626（赤） | #9CA3AF（Gray 400） |
| 角丸 | 8dp | 6dp |
| アクションボタンとの距離 | 4dp（隣接） | 中央のExpanded分だけ離れる（80dp以上） |

### REQ-VWA-04: 状態バッジ仕様

| 項目 | Before | After |
|---|---|---|
| フォントサイズ | 10px | 11px |
| フォントウェイト | normal | 600 (semibold) |
| padding | 2px 6px | 3px 8px |
| 角丸 | 4dp | 6dp |
| 配置 | コンテンツColumn末尾 | 左カラム・日付の下 |

- 状態バッジのタップ機能（アクションボタンと同機能）はスコープ外（将来検討）

### REQ-VWA-05: 「休憩」アクションの削除（バグ修正）

- `TopicConfig` の `visitWork` の `markActions` から「休憩」（`toggleBreak`）を除去する
- 理由: `toggleBreak` は ActionTime の専用アクションとして既に存在しており、markActions に含まれていると機能重複が発生する

### REQ-VWA-06: アクション表示順の変更（バグ修正）

- visitWork トピックのアクション表示順を以下に変更する:
  1. 到着
  2. 作業開始
  3. 作業終了
  4. 出発
- 現状の表示順が作業フローと一致していないため修正する

- 削除ボタンの既存の確認ダイアログ動作は維持する

## スコープ外

- 状態バッジのタップ機能（アクションボタンと同機能）: 未確定のため将来検討
- movingCost / travelExpense トピックのカードレイアウト変更
- Link（区間）カードの変更
- ダークテーマ対応
- スワイプ削除（案A）・長押しコンテキストメニュー（案B）の導入

## 受け入れ条件

- [ ] visitWork トピックの Mark カードが中央配置レイアウト（左: 日付+バッジ / 中央: 地点名+アクションボタン / 右: 削除ボタン）で表示される
- [ ] アクションボタンがカード中央に大きく表示される（テキスト「アクション」付き、高さ36dp）
- [ ] 削除ボタンがカード右端に小さく（28x28dp）・薄く（opacity 0.6）・グレー系（背景 #F3F4F6 / アイコン #9CA3AF）で表示される
- [ ] アクションボタンと削除ボタンが十分な距離（80dp以上）で離れている
- [ ] 状態バッジが左カラムに配置され、11px / semibold で表示される
- [ ] visitWork の markActions から「休憩」が除去されている
- [ ] アクション表示順が「到着 -> 作業開始 -> 作業終了 -> 出発」の順になっている
- [ ] movingCost / travelExpense トピックのカードレイアウトが変更されていない
- [ ] 既存の削除確認ダイアログが正常に動作する
- [ ] 既存の ActionTime 起動・状態遷移が正常に動作する
- [ ] `dart analyze` エラー・警告 0

## 備考

- 削除ボタンの色は、赤（#DC2626）が目立ちすぎるというユーザーフィードバックを受け、グレー系（背景 #F3F4F6 / アイコン #9CA3AF）を定義した。削除という操作の性質上、完全に目立たなくするのではなく「控えめだが存在は分かる」レベルを狙う
- レイアウト変更は `topicConfig.showActionTimeButton == true` の分岐で適用し、false の場合は従来レイアウトを維持する
- デザイン叩き（`docs/Design/draft/visit_work_action_button_center_design.html`）を実装の視覚参照とする
- REQ-VWA-05（休憩アクション削除）と REQ-VWA-06（表示順変更）は TopicConfig の修正のみで対応可能と想定
