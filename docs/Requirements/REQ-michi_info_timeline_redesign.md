# 要件書: MichiInfo タイムライン UI 再設計

要件ID: REQ-michi_info_timeline_redesign
作成日: 2026-04-07
ステータス: 作成中

---

## 背景・目的

MichiInfo の「ミチ」タブのタイムライン表示を以下の観点で改善する。

1. Mark カードとドットの接続表現を吹き出し（三角ポインター）から罫線に変更し、視覚的にシンプルかつ整合感のある表示にする
2. タイムライン縦線をカード高さに合わせて短縮し、行間の余白感を改善する
3. 距離表示をマーク間スパン矢印形式に変更し、Mark 間・区間間・混在パターンの距離を直感的に把握できるようにする

---

## ユーザーストーリー

ドライブ記録を見返すとき、各地点（Mark）間の距離と区間（Link）の距離を一目で把握したい。
マーク間に区間が挟まる場合も、合計距離と各区間の内訳が並んで見えることで全体像を掴みやすくなる。

---

## 変更要件

### 変更1: Markカードとドットの接続を罫線に変更

- **現状**: Mark カードの左端から縦線ドットへの三角形ポインター（吹き出し型）で接続
- **変更後**: 三角形を廃止し、カード左端からドットへの水平罫線（横線）に変更
- カードとドットが線でつながっているように見える
- 三角形の分だけあったカード左余白は罫線の接続点に合わせて見直す

### 変更2: タイムライン縦線の長さをカード高さに合わせて短縮

- **現状**: 縦線が行全体の高さ（上下端まで）で描画されている
- **変更後**: 縦線はカード行の上下端まで描画する。行間のパディングには縦線を描画しない
  - 行の上端〜ドット上端（Mark の場合）またはカード上端（Link の場合）
  - ドット下端（Mark の場合）またはカード下端（Link の場合）〜行の下端（`_cardHeight` まで）
  - 行間の余白（`ListView` のアイテム間スペース）は縦線なし

### 変更3: 距離表現をマーク間スパン矢印形式に変更

#### 3-1. カード幅の変更

- Mark カードの横幅を Link カードより広くする
  - Mark カード: 利用可能幅の広い領域を使用（右側距離エリアにかぶらない範囲で最大化）
  - Link カード: Mark カードより短い幅とし、右側に距離表示エリアを確保する

#### 3-2. 距離表示の4パターン

距離表示エリアは画面右側に配置する。表示パターンは以下の通り。

##### パターン1：マーク間（Link なし）

```
[Mark カード（幅広）        ]
                              ↕ 80 km
[Mark カード（幅広）        ]
```

- Mark 行の右側に両方向矢印と距離を表示
- 矢印は1行内（Mark 行の右側エリア）に収まる高さ
- 距離: 当該 Mark から次の Mark までのメーター差分

##### パターン2：区間間（Mark なし）

```
[Link カード（短め）   ] ↕ 80 km
[Link カード（短め）   ] ↕ 80 km
```

- 各 Link 行の右側に、その Link の区間距離を表示
- 矢印と距離は Link 行内に収まる

##### パターン3：マーク-区間-マーク

```
[Mark カード（幅広）        ]  ↑
[Link カード（短め）   ] ↕ 79km | 80 km
[Mark カード（幅広）        ]  ↓
```

- 右側遠方（最右列）に Mark 間スパン矢印と合計距離を表示（複数行にまたがる）
- Link 行の右側（スパン矢印の左）に区間距離を小さく表示
- スパン矢印の距離: Mark 間のメーター差分（合計）
- Link の右距離: その Link の区間距離

##### パターン4：マーク-区間-区間-マーク

```
[Mark カード（幅広）        ]        ↑
[Link カード（短め）   ] ↕ 79km | ---内---| 160 km
                                  80 km  |
[Link カード（短め）   ] ↕ 79km |        |
[Mark カード（幅広）        ]        ↓
```

- 最右列に Mark 間スパン矢印と合計距離（メーター差分）
- 中間列に「---内---」区切りと各区間の内訳合計
- 各 Link 行の右に個別区間距離
- 区間が3件以上の場合も同様のパターンを継続適用

#### 3-3. 表示値の定義

| 表示値 | 元データ | 表示場所 |
|---|---|---|
| Mark 間スパン距離（合計） | `MarkLinkItemProjection.displayMeterDiff`（既存） | 最右列スパン矢印横 |
| 区間距離（個別） | `MarkLinkItemProjection.displayDistanceValue`（既存） | Link 行右の小矢印横 |

- 既存の `displayMeterDiff`・`displayDistanceValue` フィールドはそのまま利用する
- Projection 層・BLoC 層の変更は原則不要

#### 3-4. 凡例（_DistanceLegend）の更新

- 現状の凡例「メーター差分 / 区間距離」を新しい表示形式に合わせて見直す
  - スパン矢印 = メーター差分（Mark 間合計）
  - Link 右の小矢印 = 区間距離（Link 個別）

## スコープ外（本要件では変更しない）

- MichiInfoBloc / Event / State / Delegate の構造

- Mark アクションボタン（`_MarkActionButtons`）
- FAB（地点追加・区間追加）
- ダークテーマ対応
- アニメーション
- 距離単位の切り替え

## 受け入れ条件

- [ ] Mark カードの左端とドットが三角形ではなく水平罫線で接続されている
- [ ] タイムライン縦線がカード高さの範囲内に収まっている
- [ ] Mark カードが Link カードより横幅が広い
- [ ] パターン1（Mark-Mark）: Mark 行右側にスパン矢印と距離が表示される
- [ ] パターン2（Link-Link）: 各 Link 行右側に区間距離が表示される
- [ ] パターン3（Mark-Link-Mark）: 最右列にスパン矢印（合計）、Link 行右に区間距離が表示される
- [ ] パターン4（Mark-Link-Link-Mark）: 最右列にスパン矢印（合計）、各 Link 行右に区間距離が表示される
- [ ] 既存の Mark / Link タップ操作が引き続き動作する
- [ ] MarkDetail / LinkDetail 保存後に一覧の表示が更新される

## 補足・デザイン注記

- 画面右側の距離表示エリアは固定幅を2段構造（Link 個別距離列 ＋ スパン矢印列）とする構成が候補
- スパン矢印はリスト内の Mark から次の Mark まで複数行にまたがる描画が必要
  - ListView 内での複数行スパン描画は実装難易度が高いため、architect が実装方針を検討すること
- パターン判定ロジック（連続する Mark 間に何件の Link があるか）は Widget 層 or Projection 層で計算する
  - どちらが適切かは architect が判断すること
