# 要件書: B-20 訪問作業シードデータにActionTime情報追加

## 概要

訪問作業トピックのシードデータ（シナリオC: 横浜エリア訪問ルート）にActionTime情報が含まれていない。各Markに到着・出発・作業開始・作業終了・休憩などのActionTimeLogを追加し、ダッシュボードの作業記録グラフにもデータが反映されるようにする。

---

## ユーザーストーリー

> 外回り営業担当者として、訪問作業のサンプルデータから「どのようにアクションを記録するか」をひと目で理解したい。
> 具体的なタイムラインが入っているサンプルがあれば、自分の使い方のイメージが湧く。

> ダッシュボードを開いたとき、グラフにサンプルデータが反映されていると、集計機能の価値がすぐに伝わる。

---

## 背景

- シナリオCには Mark・Link・支払いは存在するが、ActionTimeLog（アクション記録）が0件
- 訪問作業トピックの主要機能は「到着/出発/作業開始/作業終了/休憩」のアクション記録
- サンプルデータにActionTimeがないと、タイムラインタブが空白になり機能の説明にならない
- ダッシュボードの作業記録グラフもActionTimeをもとに集計されるため、グラフが空のまま

---

## スコープ

### 対象

- `seed_data.dart` のシナリオC部分にActionTimeLogデータを追加する
- 対象Markは以下の3件（事務所出発・帰着はアクション記録なし）:
  - A社（横浜駅前）: arrive → start → end
  - B社（みなとみらい）: arrive → start → break → end
  - C社（磯子）: arrive → start → end

### 対象外

- ActionTimeLog の業務ロジック・Domainモデルは変更しない
- シナリオA・シナリオBのシードデータは変更しない
- ダッシュボードの集計ロジック自体は変更しない

---

## ActionTimeログ設計

> 基準日: インストール日 -3日（シナリオCのイベント日と同じ）

### A社（横浜駅前）

| アクション | 時刻 |
|---|---|
| arrive（到着） | 09:15 |
| start（作業開始） | 09:20 |
| end（作業終了） | 10:45 |

作業時間: 1時間25分

### B社（みなとみらい）

| アクション | 時刻 |
|---|---|
| arrive（到着） | 11:05 |
| start（作業開始） | 11:10 |
| break（休憩開始） | 12:00 |
| start（作業再開） | 13:00 |
| end（作業終了） | 14:20 |

作業時間: 2時間10分（休憩1時間を除く実働）

### C社（磯子）

| アクション | 時刻 |
|---|---|
| arrive（到着） | 14:50 |
| start（作業開始） | 14:55 |
| end（作業終了） | 16:10 |

作業時間: 1時間15分

---

## 実装方針

- ActionTimeLogの日時はシナリオCのイベント基準日（インストール日 -3日）に上記の時刻を組み合わせて生成する
- 固定時刻を使用し、インストール日に対して相対的に計算する
- ダッシュボードの集計はActionTimeログを読んで自動集計されるため、データ追加のみで反映される

---

## 受け入れ条件

- [ ] シナリオCのA社・B社・C社の各Markにそれぞれ所定のActionTimeLogが登録されること
- [ ] イベント詳細のタイムラインタブにActionTimeのログが時系列で表示されること
- [ ] 概要タブの作業時間サマリーに合計作業時間（約4時間50分）が表示されること
- [ ] ダッシュボードの作業記録グラフにシナリオCのデータが反映されること
- [ ] 修正後のIntegration Testが全件PASSすること
