# 要件書: MichiInfo カード間挿入機能

- **要件ID**: REQ-michi_info_card_insert
- **作成日**: 2026-04-08
- **担当**: product-manager
- **ステータス**: 確定

---

## 背景・目的

MichiInfo タイムラインは現状、カードを末尾に追加するのみで、既存カードの間に挿入する手段がない。
ドライブ記録の性質上、途中で気づいた地点やリンクを後から正しい順序に挿入したいニーズがある。
本要件はタイムラインへの中間挿入 UI を Pattern 3（FAB型）で実装することを定義する。

---

## 採用パターン: Pattern 3「FAB型 ＋ 挿入モード ＋ ボトムシート選択」

デザイン提案: `docs/Design/2026-04-07_marklink_insert_button_proposal.html`

### UXフロー

1. **通常モード**: 画面右下に Amber FAB（`+` アイコン）を表示
2. **挿入モード突入**: FAB タップ → タイムライン上の各カード間にインジケーター（行）が出現
3. **挿入ポイント選択**: インジケーターをタップ → 挿入位置が確定
4. **種別選択**: ボトムシートで「Mark を追加」「Link を追加」の2択を表示
5. **詳細入力**: 既存の MarkDetail / LinkDetail 画面へ遷移（新規作成フロー）
6. **保存後**: 指定した位置に新しいカードが挿入されてタイムラインに反映

### キャンセル

- 挿入モード中に FAB（またはキャンセルボタン）を再タップ → 通常モードに戻る
- ボトムシートをスワイプで閉じる → 挿入ポイント選択に戻る（挿入モード継続）

---

## カラー定義

| 用途 | カラー | 備考 |
|---|---|---|
| FAB 背景 / インジケーター | `#F59E0B`（Amber） | Mark（Teal）・Link（Emerald）と区別する第3色 |
| FAB アイコン | `#FFFFFF`（白） | WCAG AA 対応のためアイコンのみ白 |
| インジケーター線 | `#F59E0B` 40% alpha | タイムラインを圧迫しない薄い表示 |

---

## 順序管理の方針

- 現在の `sortOrder`（または同等フィールド）を使って順序を管理する
- 挿入時は前後カードの `sortOrder` の中間値を割り当てる
- 詳細な実装方針は architect に委ねる

---

## 対象外（スコープ外）

- カードの並び替え（ドラッグ＆ドロップ）
- 挿入後のカード削除・移動
