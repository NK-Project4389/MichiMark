# 要件書: B-17 本番シードデータ見直し

## 概要

テストデータが本番配信されているため、新規インストール時に表示されるシードデータをユーザーが入力イメージしやすいサンプルデータに差し替える。

- 全クリアはしない（アプリの使い方が伝わるサンプルを残す）
- シナリオA・B・C の3イベントを用意する

## 実装方針

- 既存テストシードデータを上記3イベントに**置き換える**
- DBマイグレーション不要（アプリ初回起動時のシード投入コードを差し替え）
- 日付はアプリインストール日から相対的に設定（固定日付にしない）
  - シナリオA: インストール日 -7日
  - シナリオB: インストール日の当月1日〜
  - シナリオC: インストール日 -3日

## 受け入れ条件

- [ ] 新規インストール時に3イベントが表示される
- [ ] 各イベントのタイムライン・集計・支払いタブが正しく表示される
- [ ] シナリオBの集計タブで走行距離合計・推計燃費が表示される
- [ ] シナリオCの概要タブで作業時間サマリーが表示される
- [ ] 既存ユーザーへの影響なし（初回起動フラグで制御）
