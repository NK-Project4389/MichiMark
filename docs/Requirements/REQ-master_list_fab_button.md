# 要件書: マスター項目一覧のFABボタン配置変更

- **要件ID**: REQ-master_list_fab_button
- **作成日**: 2026-04-16
- **担当**: product-manager
- **ステータス**: 確定
- **種別**: UI改善（UI-21）

---

## 1. 背景・目的

交通手段・メンバー・タグ・アクションの各マスター項目一覧画面では、新規追加ボタンをAppBar右側のIconButtonとして配置していた。
MichiInfo等の他画面では同操作をScaffoldのFAB（FloatingActionButton）として右下に配置しており、UI上の一貫性が失われている。
本要件では操作ボタンの配置を統一し、ユーザーが各画面で同じ操作パターンを直感的に使えるようにする。

---

## 2. ユーザーストーリー

> マスター項目（交通手段・メンバー・タグ・アクション）を管理するユーザーとして、
> 一覧画面の右下にある「＋」ボタンをタップして新規項目を追加したい。
> なぜなら、他の一覧画面と同じ操作パターンで使えると迷わずに操作できるから。

---

## 3. 要件一覧

### 3.1 FABの追加

- 各マスター項目一覧画面の `Scaffold.floatingActionButton` に「＋」FABを配置する
- FABのスタイルは既存のMichiInfo FABと同一とする

| 項目 | 値 |
|---|---|
| アイコン | `Icons.add` |
| 背景色 | `Color(0xFFF59E0B)`（オレンジ・テーマカラー） |
| アイコン色 | 白（`Colors.white`） |
| タップ動作 | 従来のAppBar「＋」ボタンと同じ新規作成処理 |

### 3.2 AppBar右側の「＋」ボタン削除

- AppBarの `actions` に配置している `IconButton`（Icons.add）を削除する
- AppBar自体・タイトル・その他のactionsは変更しない

### 3.3 対象画面

| 画面 | ファイル（想定） |
|---|---|
| 交通手段設定 | `trans_setting_page.dart` |
| メンバー設定 | `member_setting_page.dart` |
| タグ設定 | `tag_setting_page.dart` |
| アクション設定 | `action_setting_page.dart` |

---

## 4. 影響範囲

| 画面 | 変更内容 |
|---|---|
| 交通手段設定（TransSettingPage） | AppBar actionsの「＋」削除・FAB追加 |
| メンバー設定（MemberSettingPage） | AppBar actionsの「＋」削除・FAB追加 |
| タグ設定（TagSettingPage） | AppBar actionsの「＋」削除・FAB追加 |
| アクション設定（ActionSettingPage） | AppBar actionsの「＋」削除・FAB追加 |

---

## 5. 対象外

- FABタップ後の新規作成ロジック・画面遷移の変更
- 各マスター項目の入力フォームの変更
- 一覧のリストアイテム・編集・削除機能の変更
- MichiInfo等、既にFABを使用している画面への変更

---

## 6. 受け入れ条件

1. 各マスター項目一覧画面を開いたとき、AppBar右側に「＋」アイコンが表示されない
2. 各マスター項目一覧画面を開いたとき、画面右下にオレンジ色のFAB（「＋」アイコン）が表示される
3. FABをタップしたとき、従来のAppBar「＋」ボタンと同じ新規作成画面が開く
4. FABのスタイル（色・アイコン）がMichiInfo等の既存FABと統一されている
5. 上記4画面すべてで条件1〜4を満たす
