# 要件書: メンバー選択 全選択 / 全解除ボタン追加

- **要件ID**: REQ-member_select_all_clear
- **作成日**: 2026-04-13
- **担当**: product-manager
- **ステータス**: 確定
- **種別**: UI改善（UI-11）

---

## 1. 背景・目的

MarkDetail・LinkDetail のメンバー選択、PaymentDetail の割り勘メンバー選択にて、メンバー数が多い場合に1件ずつ操作する手間がかかる。

「全選択」と「全解除」ボタンを追加し、一括操作を可能にする。

---

## 2. 要件一覧

### 2.1 対象画面

| 対象画面 | 対象セクション |
|---|---|
| MarkDetail（地点詳細） | メンバー選択インラインチップ |
| LinkDetail（区間詳細） | メンバー選択インラインチップ |
| PaymentDetail（支払詳細） | 割り勘メンバー選択インラインチップ |

### 2.2 ボタン仕様

| ボタン名 | 動作 |
|---|---|
| 「全選択」| 全メンバーを選択状態にする |
| 「全解除」| 全メンバーを非選択にする（制約あり → 2.3参照） |

### 2.3 支払者の制約（PaymentDetail のみ）

- PaymentDetail において、「支払者（payer）」に設定されているメンバーは「全解除」操作対象外とする
- 「全解除」タップ時：支払者以外の全メンバーを非選択にする
- 支払者は常に選択状態を維持する（既存の仕様に準ずる）

### 2.4 MarkDetail / LinkDetail の全解除

- 支払者制約なし
- 全員を非選択にできる（既存のメンバー未選択ガードと連動：保存時にバリデーションエラーが出る仕様は変更しない）

### 2.5 ボタン配置

- メンバー選択セクションのヘッダー行（例：「メンバー」ラベルの右隣）に横並びで配置
- またはメンバーチップ一覧の上部に配置
- デザインは既存のインラインUI（チップスタイル）に合わせる
- ボタンサイズは小さめ（テキストボタンまたはアウトラインボタン）
