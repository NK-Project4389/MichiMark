# 要件書: MichiInfo / PaymentInfo 削除確認ダイアログ追加

- **要件ID**: REQ-delete_confirmation_dialog
- **作成日**: 2026-04-13
- **担当**: product-manager
- **ステータス**: 確定
- **種別**: UI改善（UI-7）

---

## 1. 背景・目的

現在 MichiInfo（Mark / Link カード）および PaymentInfo（伝票カード）の削除アイコンをタップすると、確認なしに即座に削除される。

誤操作によるデータ消失を防ぐため、削除ボタン押下時に確認ダイアログを表示する。

---

## 2. 要件一覧

### 2.1 対象画面・操作

| 対象 | 削除操作のトリガー |
|---|---|
| MichiInfo → Markカード | 右端の赤ゴミ箱アイコンタップ |
| MichiInfo → Linkカード | 右端の赤ゴミ箱アイコンタップ |
| PaymentInfo → 伝票カード | 右端の赤ゴミ箱アイコンタップ |

### 2.2 確認ダイアログ仕様

| 項目 | 内容 |
|---|---|
| ダイアログ種別 | iOS ネイティブスタイル（`showCupertinoDialog` または `AlertDialog`）|
| タイトル | 「削除しますか？」|
| メッセージ | 「この操作は取り消せません。」|
| ボタン（確定） | 「削除」（破壊的操作として赤色） |
| ボタン（キャンセル） | 「キャンセル」|
| 「削除」タップ時 | 既存の削除処理を実行 |
| 「キャンセル」タップ時 | ダイアログを閉じるだけ（削除しない） |

### 2.3 ダイアログ表示のタイミング

- ゴミ箱アイコンタップ → ダイアログ表示 → 「削除」タップ → 既存の削除イベントを dispatch

---

## 3. 影響範囲

| 対象コンポーネント | 変更内容 |
|---|---|
| MichiInfo View（MichiInfoView）| ゴミ箱アイコンのonTap に確認ダイアログを挿入 |
| PaymentInfo View（PaymentInfoView）| ゴミ箱アイコンのonTap に確認ダイアログを挿入 |
| MichiInfo Bloc / PaymentInfo Bloc | 変更なし（既存の削除イベントをそのまま使用） |

---

## 4. 対象外

- 削除後の処理（既存実装に準ずる）
- 削除取り消し（Undo）
- 画面遷移・アニメーション変更
