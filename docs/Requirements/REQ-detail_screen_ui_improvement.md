# 要件書: MarkDetail/LinkDetail/PaymentDetail UI改善

- **要件ID**: REQ-detail_screen_ui_improvement
- **作成日**: 2026-04-12
- **担当**: product-manager
- **ステータス**: 確定
- **種別**: UI改善（UI-5）

---

## 1. 背景・目的

MarkDetail・LinkDetail・PaymentDetailの3画面について、操作性とUIの一貫性を改善する。
戻るボタンをキャンセルボタンに置き換え、ヘッダのタイトル形式を統一し、操作ボタンの配置を整理する。

---

## 2. 要件一覧

### 2.1 戻るボタンの廃止

- ナビゲーションバーの戻るボタン（`<` アイコン等）を廃止する
- 代わりに「キャンセル」ボタンを画面内に配置する（§2.3参照）

### 2.2 ヘッダタイトルの形式変更

| 画面 | 変更前 | 変更後 |
|---|---|---|
| MarkDetail | 名称のみ（例：`〇〇SA`） | `地点詳細：〇〇SA` |
| LinkDetail | 名称のみ | `区間詳細：〇〇` |
| PaymentDetail | 名称のみ | `支払詳細：〇〇` |

### 2.3 キャンセル・保存ボタンの配置変更

| 項目 | 変更前 | 変更後 |
|---|---|---|
| キャンセル | なし（戻るボタンで代替） | 保存ボタンの左に配置 |
| 保存 | ナビゲーションバー内 | フォームの最下部（左右中央寄せ・横並び） |
| 配置位置 | ナビゲーションバー | フォームの最下部 |
| 並び | — | キャンセル（左）・保存（右） |

- 左右中央寄せ（画面横幅に対してセンタリング）
- キャンセルタップ時：変更を破棄して画面を閉じる

---

## 3. 影響範囲

| 画面 | 変更内容 |
|---|---|
| MarkDetail | 戻るボタン廃止・ヘッダ「地点詳細：(名称)」・キャンセル/保存を最下部に配置 |
| LinkDetail | 戻るボタン廃止・ヘッダ「区間詳細：(名称)」・キャンセル/保存を最下部に配置 |
| PaymentDetail | 戻るボタン廃止・ヘッダ「支払詳細：(名称)」・キャンセル/保存を最下部に配置 |

---

## 4. 対象外

- 各画面の入力フィールドの変更
- 保存・削除ロジックの変更
