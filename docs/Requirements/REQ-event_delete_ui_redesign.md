# 要件書: イベント削除UI変更

- **要件ID**: REQ-event_delete_ui_redesign
- **作成日**: 2026-04-12
- **担当**: product-manager
- **ステータス**: 確定
- **種別**: UI改善（UI-1）

---

## 1. 背景・目的

イベント一覧のカードスワイプ削除を廃止し、イベント詳細画面のヘッダに削除アイコンを移動する。

誤削除防止と操作の一貫性向上が目的。

---

## 2. 要件一覧

| 項目 | 内容 |
|---|---|
| 廃止する操作 | イベント一覧画面のスワイプ削除ボタン表示（`flutter_slidable`） |
| 追加する操作 | イベント詳細画面のヘッダ右側に削除アイコン＋「イベント削除」ラベルを追加 |
| 確認ダイアログ | **あり**（削除アイコンタップ → 確認ダイアログ表示 → 確認後に削除） |
| 削除ロジック | 変更なし（既存のカスケード削除仕様に準ずる） |

---

## 3. 確認ダイアログ仕様

| 項目 | 内容 |
|---|---|
| タイトル | 「イベントを削除しますか？」 |
| メッセージ | 「このイベントに関連するすべての情報が削除されます。」 |
| ボタン | 「削除」（破壊的アクション）／「キャンセル」 |

---

## 4. 影響範囲

| 画面 | 変更内容 |
|---|---|
| イベント一覧画面 | `flutter_slidable` によるスワイプ削除UI撤去 |
| イベント詳細画面 | ヘッダ右側に削除アイコン＋「イベント削除」ラベル追加・確認ダイアログ追加 |

---

## 5. 対象外

- イベント削除時のカスケード削除仕様（既存実装に準ずる）
- 削除取り消し（Undo）

---

## 6. テストシナリオ

| TC-ID | シナリオ | 期待結果 |
|---|---|---|
| TC-EDR-001 | イベント一覧でカードを左スワイプする | 削除ボタンが表示されない |
| TC-EDR-002 | イベント詳細画面のヘッダ右側を確認する | 削除アイコンが表示されている |
| TC-EDR-003 | 削除アイコン付近を確認する | 「イベント削除」ラベルが表示されている |
| TC-EDR-004 | 削除アイコンをタップする | 確認ダイアログが表示される |
| TC-EDR-005 | 確認ダイアログで「キャンセル」をタップする | ダイアログが閉じ、イベントは削除されない |
| TC-EDR-006 | 確認ダイアログで「削除」をタップする | イベントが削除されて一覧画面に戻る |
