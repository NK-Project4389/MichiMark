# 要件書: B-19 訪問作業シードデータから区間データ削除・IntTest見直し

## 概要

訪問作業（visitWork）トピックでは区間（Link）を作成しない仕様であるにもかかわらず、現在のシードデータ（シナリオC: 横浜エリア訪問ルート）にはLink 4件が含まれている。シードデータからLinkデータを削除し、整合性を取る。

---

## ユーザーストーリー

> 外回り営業担当者として、訪問作業のサンプルデータが正しい仕様を反映していることを期待する。
> 区間データは存在しないはずなのに表示されると混乱する。

---

## 背景・問題

- シナリオCのシードデータは B-17 で導入されたが、誤って区間（Link）データ4件が含まれた状態で実装された
- 訪問作業トピックの仕様では「区間を作成しない」とされている
- このままだとサンプルデータがユーザーに誤った操作イメージを与える
- また、Integration Test のスルー検証もLink件数を前提にしている可能性があるため合わせて見直す

---

## スコープ

### 対象

- `flutter/lib/` 配下の `seed_data.dart`（シナリオC部分）のLink生成コードを削除する
- Integration Test でシナリオCのLink件数・Link表示を検証している項目を見直す・削除する

### 対象外

- シナリオA（ドライブ）・シナリオB（移動コスト）のシードデータは変更しない
- 訪問作業トピックの業務ロジック・Domainモデル自体は変更しない
- 既存ユーザーのデータには影響しない（シード投入は初回起動時のみ）

---

## 実装方針

- `seed_data.dart` のシナリオC内でLinkを生成している箇所をすべて削除する
- Link削除後もMark（地点）5件・支払い3件のデータは維持する
- Integration Test の該当アサーション（Link件数確認・Link表示確認）を削除または修正する

---

## 受け入れ条件

- [ ] シナリオCのシードデータにLinkが0件であること
- [ ] シードデータ投入後、横浜エリア訪問ルートのイベント詳細でLink一覧が空であること
- [ ] Mark（地点）5件・支払い3件は従来どおり表示されること
- [ ] 修正後のIntegration Testが全件PASSすること
