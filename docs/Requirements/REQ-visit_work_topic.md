# 要件書：F-3 訪問作業トピック（visitWork）

作成: 2026-04-15
担当: product-manager
タスク: T-401

---

## 概要

個人事業主が訪問作業を記録・集計するためのトピック種別 `visitWork` を追加する。
既存の Mark・ActionTimeLog・PaymentInfo をそのまま流用し、
訪問作業に特化したアクション定義・状態遷移インタープリター・集計表示を新規追加する。

---

## ユーザーストーリー

- 訪問作業者として、出発から到着・作業・出発までの時間経過をアクションで記録したい
- 1つの訪問先（Mark）にすべてのアクションをまとめて記録したい
- 作業後に「作業時間・移動時間・売上・時給換算」を確認したい
- 請求書作成や単価交渉の根拠となる時間・金額データをアプリで把握したい

---

## スコープ

### 新規追加

| 項目 | 内容 |
|---|---|
| トピック種別 | `visitWork`（訪問作業） |
| 専用アクション | 到着 / 出発 / 作業開始 / 作業終了 / 休憩（トグル） |
| 状態遷移インタープリター | ActionTimeLog → ステータス遷移の解釈ロジック |
| MichiInfo 集計 | プログレスバー（時間軸タイムライン）＋ 時間サマリー＋売上サマリー |

### 既存機能の流用（変更なし）

| 機能 | 流用方法 |
|---|---|
| Mark | 訪問先地点として使用（メーターフィールド含む） |
| ActionTimeLog | 既存の仕組みを使用。訪問作業専用アクションのみ登録する |
| PaymentInfo | 売上として記録（PaymentDetail登録フローはPOST-1/F-5で対応） |

### スコープ外

- 区間（Link）の記録（訪問作業トピックでは不要）
- メンバーの記録（不要）
- 地図上の位置情報連携（既存Markの仕様に従う）

---

## 訪問作業アクション定義

訪問作業専用のアクションマスターデータをシードとして追加する（設定画面から編集可能）。

| アクション名 | 役割 | 遷移 |
|---|---|---|
| 到着 | 移動から現場に着いた | `moving` → `staying` |
| 出発 | 現場から移動を開始する | `staying` → `moving` |
| 作業開始 | 滞在中に作業を始める | `staying` → `working` |
| 作業終了 | 作業を終えて滞在状態に戻る | `working` → `staying` |
| 休憩 | トグル（ON/OFF） | ON: `任意` → `break` / OFF: `break` → 直前の状態 |

---

## 状態（ステータス）定義

ActionTimeLog の記録を解釈して導出するステータス。Domainには保存せずインタープリターで動的に算出する。

| ステータス | 説明 |
|---|---|
| `moving` | 移動中（出発後・到着前） |
| `staying` | 現地滞在中（作業外） |
| `working` | 作業中 |
| `break` | 休憩中 |
| `unknown` | 初期状態またはActionTimeがない状態 |

---

## 休憩トグルのロジック

休憩は他のアクションと異なりトグル式で動作する。

**休憩 ON:**
1. 現在のステータスを `prevStatus` として記録（ActionTimeLogに付加情報として保持）
2. `break` ステータスへ遷移する ActionTimeLog を追加する

**休憩 OFF:**
1. `prevStatus` に戻る ActionTimeLog を追加する
2. UI上の休憩トグルをOFFにする

**休憩ON中に別のアクション（到着・出発・作業開始・作業終了）を記録した場合:**
1. 自動的に「休憩 OFF」を先に記録する
2. 続けて新しいアクションを記録する
3. UIの休憩トグルをOFFに変更する

## 集計表示（MichiInfo集計タブ）

### プログレスバー（時間軸タイムライン）

```
[出発 09:00]──移動──[到着 09:30]──作業──[休憩]──作業──[出発 13:00]
   ██移動██       █████作業█████  █休憩█  ████作業███
```

- 横軸: 時間（最初のActionTime〜最後のActionTime）
- 色分け:
  - 移動: グレー
  - 滞在: ブルー
  - 作業: テーマカラー（Teal系）
  - 休憩: オレンジ

**末端（開始・終了）:**
- 開始: 最初に記録したActionTimeの時刻
- 終了: 最後に記録したActionTimeの時刻（または「進行中」表示）

### 時間サマリー

| 項目 | 内容 |
|---|---|
| 移動時間 | `moving` ステータスの合計時間 |
| 作業時間 | `working` ステータスの合計時間 |
| 滞在時間 | `staying` ステータスの合計時間（作業外） |
| 休憩時間 | `break` ステータスの合計時間 |
| 在現地時間 | 最初の「到着」〜最後の「出発」の合計時間 |

### 売上サマリー

| 項目 | 内容 |
|---|---|
| 売上合計 | PaymentInfoの合計金額（travelExpenseの収支と同じ仕組み） |
| 時給換算 | 売上合計 ÷ 作業時間（作業時間0の場合は非表示） |

> **時給換算の価値:** 個人事業主が「実際に稼いだ単価」を把握し、請求書交渉・値付け根拠に活用できる。

---

## 実装フェーズ

本機能は以下の2フェーズで実装する。

| フェーズ | 内容 | 依存 |
|---|---|---|
| Phase A | トピック種別追加・アクション定義・ActionTimeLog 状態遷移・集計プログレスバー | なし |
| Phase B | MarkからPaymentDetail直接登録（売上品目の登録効率化） | POST-1/F-5完了後 |

---

## 前提・制約

- 設計憲章に従い BLoC パターンで実装する
- 状態遷移はドメイン層のインタープリターで算出し、DBには保存しない
- 既存の movingCost / movingCostEstimated / travelExpense トピックへの影響はゼロ
- REL-1（AppStoreリリース）完了済みにつき着手可能
