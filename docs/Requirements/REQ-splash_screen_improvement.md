## 1. 背景・目的

現在のスプラッシュ画面はネイティブ層（iOS: LaunchScreen.storyboard / Android: XMLテーマ）のみで構成されており、白背景に中央ロゴという最小限の表示にとどまっている。アプリのテーマカラー（Teal/Emerald系）との統一感がなく、起動時のブランド体験が弱い。

Flutter側のDartスプラッシュ画面は未実装のため、ネイティブスプラッシュからDashboard画面への遷移が唐突になっている。DI初期化やデータロードなどの起動処理中にユーザーへ視覚的なフィードバックを提供する仕組みもない。

本要件では、ネイティブスプラッシュの見た目改善と、Flutter側のカスタムスプラッシュ画面（アニメーション付き）の導入により、起動体験を向上させる。

## 3. 要件項目

### 3-1. ネイティブスプラッシュ（フェーズ1: OS起動直後）

ネイティブスプラッシュはOSがアプリプロセスを起動してからFlutterエンジンが描画可能になるまでの間に表示される。

- [ ] **背景色をテーマカラーに変更** — 現在の白背景をアプリロゴのTealカラー（`#2B7A9E` 相当）に統一する
- [ ] **ロゴ画像の更新** — 新規作成する白抜きロゴを設定する
- [ ] **flutter_native_splash パッケージの導入** — iOS/Androidのネイティブスプラッシュ設定をFlutter側から一元管理する
- [ ] **解像度対応** — iOS（1x/2x/3x）・Android（mdpi〜xxxhdpi）の各解像度向けロゴアセットを用意する

### 3-2. Flutterスプラッシュ（フェーズ2: Dartエンジン起動後）

Flutterエンジンが描画可能になった後、DI初期化・データロード等の起動処理が完了するまでの間に表示する。

- [ ] **カスタムスプラッシュ画面の新設** — Dart層でスプラッシュ画面ウィジェットを実装する
- [ ] **背景色** — アプリロゴのメインカラー（Teal系: `#2B7A9E` 相当）を使用。ネイティブスプラッシュと同色でシームレスに繋ぐ
- [ ] **ロゴアニメーション** — ロゴのフェードイン + スケールアップを組み合わせた起動アニメーション（1〜2秒程度）
- [ ] **表示コンテンツ** — ロゴのみ表示（アプリ名テキストなし）。ロゴはアプリアイコン画像（Icon-App-1024x1024）を使用
- [ ] **白抜きロゴ** — Teal背景上でロゴが映えるよう、新規に白抜きバリエーションを作成する
- [ ] **初期化処理の待機** — DI（GetIt）の登録完了・必要なデータのロードが終わるまでスプラッシュを表示し続ける
- [ ] **最低表示時間の設定** — 最低1秒間は表示を維持する（ロード完了が早くてもスプラッシュを継続表示）
- [ ] **遷移アニメーション** — スプラッシュ画面からDashboard画面への遷移をフェードアウトで行う

### 3-3. ルーティング・テスト互換性

- [ ] **初期ルートの変更** — 初期ルートをスプラッシュ画面（例: `/splash`）に設定し、初期化完了後に `/dashboard` へ遷移する
- [ ] **Integration Testのスプラッシュスキップ維持** — 既存の `router.go('/')` によるスプラッシュスキップパターンが引き続き動作すること。テスト時はスプラッシュを経由せず直接Dashboard画面に遷移できること

---

## 4. 受け入れ条件

1. アプリ起動時、ネイティブスプラッシュがテーマカラー背景 + ロゴで表示される（iOS/Android両対応）
2. ネイティブスプラッシュからFlutterスプラッシュへ、背景色の変化なくシームレスに遷移する
3. Flutterスプラッシュでロゴアニメーション（フェードイン + スケール）が再生される
4. DI初期化完了後にDashboard画面へフェードアウトで遷移する
5. 初期化が高速でもアニメーションが途中で打ち切られない（最低表示時間が機能する）
6. Integration Testが既存の `router.go('/')` パターンでスプラッシュをスキップできる
7. iOS/Androidの両プラットフォームで外観が統一されている

---

## 5. 非機能要件

- スプラッシュ画面の総表示時間（ネイティブ + Flutter）は通常起動で3秒以内を目安とする
- アニメーションは `AnimationController` + `CurvedAnimation` を使用し、設計憲章に準拠した実装とする
- flutter_native_splash の設定は `pubspec.yaml` 内で完結させる（別設定ファイル不要）

## 8. 備考

- 現在のIntegration Testでは `router.go('/')` を `app.main()` より先に呼ぶことでスプラッシュをスキップしている。この仕組みはFlutterスプラッシュ導入後も維持する必要がある
- ネイティブスプラッシュ → Flutterスプラッシュ → Dashboard の2段階遷移となるため、背景色の統一が重要（白→テーマカラーの「チラつき」を防ぐ）
- `FLUTTER_TEST` 環境変数による通知初期化スキップと同様のパターンが、スプラッシュ画面のルーティングにも必要になる可能性がある
