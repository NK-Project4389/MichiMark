# 要件書: カスタム数値キーパッド（CustomNumericKeypad）

作成日: 2026-04-11
作成ロール: product-manager
参照デザイン: docs/Design/draft/custom_numeric_keypad_design.html

---

## 背景・目的

- デバイス標準キーボードはOS・機種によりレイアウトが異なり「00」キー・演算子・Draft表示などを統一的に提供できない
- ドライブ記録ユースケース（燃費計算・給油金額の単価×量など）では簡易計算ニーズが存在する
- 入力中に変更前の値（Draft original）を参照できると誤入力訂正が容易になる

---

## Phase 1（本要件の対象）

### 対象ウィジェット・使用箇所

`NumericInputRow` を使用する全箇所：
- BasicInfo（燃費・ガソリン単価）
- FuelDetail
- PaymentDetail
- MarkDetail
- LinkDetail

### 機能要件

#### 1. キーパッド表示方式
- `NumericInputRow` のフィールドタップ時に Overlay BottomSheet としてキーパッドを表示する
- システムキーボードは使用しない（`showSoftKeyboard = false`）

#### 2. キー構成（C案）

```
[C]  [7] [8] [9] [÷]
[⌫] [4] [5] [6] [×]
[.]  [1] [2] [3] [−]
[00] [0]        [+]
[        =（→確定）        ]
```

- 数字キー: 0〜9, 00
- 小数点: `.`（isDecimal = false の場合は非活性グレー表示）
- 四則演算: `+`, `−`, `×`, `÷`
- クリア: `C`（全消去）
- バックスペース: `⌫`（1文字削除）
- 実行/確定キー: `=`→`確定` トグルボタン（下部全幅）

#### 3. `=`→`確定` トグル動作

- **初期状態**: ボタンラベルは `=`
- **`=` 押下時**: 四則演算の中間式（例: `152 + 48`）を計算し結果を Display に表示する。ボタンラベルを `確定` に切り替える
- **`確定` 押下時**: 表示中の数値を確定し `onChanged` コールバックを呼び出す。キーパッドを閉じる
- **演算子なし・単純入力時**: `=` を押した場合はそのまま `確定` として動作（`onChanged` を呼び出してキーパッドを閉じる）

#### 4. Display エリア（キーパッド上部）

```
変更前: 152.8 km/L    ← Draftの変更前保存値（グレー・小さめ）
        152.8          ← 現在の入力値（大きく・太め）
```

- 「変更前」行: Draftの初期値（BottomSheet表示時の `value`）を単位付きで表示。変更がなければ非表示でも可
- 現在入力値行: リアルタイムで更新。演算中間式は現在入力値として表示（例: `152 + ▋`）

#### 5. レスポンシブサイズ

- 基準幅: 375pt（iPhone SE）
- スケール係数: `sf = MediaQuery.of(context).size.width / 375`
- キーサイズ・フォントサイズすべてを `sf` で等比スケール
- アスペクト比は機種によらず一定に保つ

#### 6. カラー（ライトモード / ダークモード）

| キー種別 | ライト背景 | ライトテキスト | ダーク背景 | ダークテキスト |
|---|---|---|---|---|
| 数字 (0-9, 00) | #FFFFFF | #1A1A2E | #2C3C3C | #E8F4F2 |
| 演算子 (+−×÷) | #D8EDEB | #2D6A6A | #1A3A3A | #4ECDC4 |
| 確定 / = | #2D6A6A | #FFFFFF | #4ECDC4 | #1A1A2E |
| ⌫ Delete | #E8EEF0 | #4A6060 | #243030 | #9BB5B3 |
| C クリア | #FFE5E5 | #C0392B | #3A1A1A | #FF6B6B |
| キーパッド背景 | #F2F4F4 | — | #1C2626 | — |

#### 7. 非変更項目

- `NumericInputRow` の外部インタフェース（label, unit, value, isDecimal, onChanged）は変更しない
- カンマ整形・生数値文字列の受け渡しロジックは現状維持

---

## Phase 2（別要件・タスクボード追記のみ）

- 四則演算の演算子キーは Phase 1 で配置するが、演算ロジックと中間式表示は Phase 2 で実装
- Phase 1 では演算子キーをタップしても無視（または非表示）
- Phase 2 要件書は Phase 1 完了後に別途作成

## Phase 3（別要件・タスクボード追記のみ）

- キーパッドを Push-up 方式（システムキーボードと同様に画面コンテンツを押し上げる形）に移行
- Phase 2 完了・実機フィードバック後に判断
