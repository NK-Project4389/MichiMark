# 要件書: UI-22 ダッシュボード移動コストグラフ ポップアップ改善

- **要件ID**: REQ-dashboard_graph_popup
- **作成日**: 2026-04-16
- **担当**: product-manager
- **ステータス**: 確定
- **種別**: UI改善（UI-22）
- **関連要件**: REQ-dashboard（F-2 ダッシュボード機能）

---

## 1. 背景・目的

ダッシュボードの移動コストグラフ（fl_chart 使用）で棒をタップしたときに表示されるポップアップ（ツールチップ）が、黒文字に濃い緑背景という配色のため視認性が低い。

また、現状のポップアップには距離・金額どちらか一方しか表示されておらず、両方の情報を一度に確認できない。

本改善では配色の修正と情報量の拡充を行い、グラフの操作性・可読性を向上させる。

---

## 2. ユーザーストーリー

> ダッシュボードを確認するユーザーとして、
> 移動コストグラフの棒をタップ・長押ししたときに見やすいポップアップで距離と金額の両方を確認したい。
> なぜなら、1回の操作で必要な情報をすべて把握できると判断が早くなるから。

---

## 3. 改善内容

### 3-1. ポップアップの配色変更（タップ時）

| 項目 | 現状 | 改善後 |
|---|---|---|
| 背景色 | 濃い緑（視認性低） | 半透明の暗色（例: `Colors.black87` 相当） |
| 文字色 | 黒（背景と低コントラスト） | 白（`Colors.white`） |
| 角丸 | 未指定 | `BorderRadius.circular(8)` 相当 |

- テーマカラーとの整合性を考慮し、Spec フェーズで最終配色を確定すること
- `BarTouchTooltipData` の `tooltipBgColor` / `getTooltipItem` で実装する

### 3-2. 長押し時のポップアップ拡充

| 操作 | 表示内容 |
|---|---|
| タップ（既存） | 日付 + 金額（円） |
| 長押し（新規） | 日付 + 走行距離（km） + 金額（円） の両方 |

- 長押し検知は fl_chart の `longPressDuration` または `onLongPressMoveUpdate` で実装する
- 長押し中はポップアップを継続表示し、指を離したら非表示に戻す

---

## 4. スコープ

### 対象

- ダッシュボード画面の移動コストグラフ（movingCost / movingCostEstimated）
- fl_chart の `BarTouchTooltipData` / `BarTouchData` 設定

### 対象外

- 作業記録（visitWork）グラフのポップアップ（別トピック・別グラフ種別）
- 旅費（travelExpense）のカレンダービュー
- 有料版の期間指定機能
- グラフのデータ取得ロジック・Adapter・Domain 層の変更

---

## 5. 受け入れ条件

| # | 条件 |
|---|---|
| AC-1 | 移動コストグラフの棒をタップしたとき、白文字・暗色背景のポップアップが表示される |
| AC-2 | タップ時のポップアップに日付と金額（円）が表示される |
| AC-3 | 移動コストグラフの棒を長押ししたとき、日付・走行距離（km）・金額（円）の3項目が表示される |
| AC-4 | 長押し中はポップアップが継続表示される |
| AC-5 | 長押しを離したらポップアップが非表示になる |
| AC-6 | ポップアップの背景色・文字色がテーマカラーと視覚的に整合している |
| AC-7 | movingCost と movingCostEstimated の両モードでポップアップが正しく動作する |

---

## 6. 非機能要件

- ポップアップの配色定数はテーマファイルまたは定数ファイルで一元管理する（Widget 内へのハードコーディング禁止）
- ポップアップ表示ロジックは Widget 層で完結させる（Bloc / Adapter 層の変更不要）

---

## 7. 対象外（現フェーズ）

- アニメーション付きポップアップ表示
- 複数棒を同時選択した際のポップアップ
- ポップアップのカスタム形状（吹き出し型など）
