# 要件書: UI-23 MichiInfo 日付区切り表示

作成日: 2026-04-17
作成者: product-manager
バージョン: 2.0
参照デザイン叩き: `docs/Design/draft/michi_info_date_separator_design.html`

---

## 概要

MichiInfo「ミチ」タブのMarkiLinkタイムラインに、日付変わり目を示す**日付区切り行**を挿入する。
前後で同じ日付のカードが連続する場合は表示せず、日付が変わるタイミングにのみ挿入する。
全トピック（訪問作業・ドライブ等）共通で適用し、道路イメージのタイムラインデザインを崩さない最小介入の実装とする。

---

## ユーザーストーリー

- ユーザーとして、タイムラインをスクロールしながら「ここから翌日」と一目で分かるようにしたい
  → 日付が変わるタイミングに水平区切り行が入ることで、旅程・作業の節目をスキャンできる
- ユーザーとして、各カードに日付を繰り返し表示されても冗長に感じたくない
  → 区切り1行にまとめることでカード自体の情報密度を維持できる
- ユーザーとして、日帰りイベントでも「いつのドライブ・作業か」を確認したい
  → 先頭アイテムの前にも区切りを1行表示する

---

## 機能要件

### REQ-DS-01: 区切り挿入ルール

- タイムラインを上から走査し、**前のカードと日付が異なる値に変わる直前**に区切りを挿入する
- **リスト最初のカードの前にも必ず区切りを挿入する**（日付が1種類のみの場合も同様）
- 挿入位置: 直前のカード（Mark/Linkカード）と次のカードの間
- InsertMode（カード挿入操作中）は区切りを非表示にする（Y座標ずれ防止の既存ロジックに合わせる）

### REQ-DS-02: 区切りの表示形式

- **日付フォーマット: `yyyy/MM/dd`**（例: `2024/08/11`）
- 日付比較は `MarkLinkItemProjection` が持つ日付フィールドの `yyyy/MM/dd` 部分のみで行う
- 同一日付が連続するカード間には区切りを表示しない

### REQ-DS-03: 区切りのビジュアル（推奨案A: スレートグレー系）

- レイアウト: `横線 ─── 日付バッジ ─── 横線`（Row構成）
- 横線色: `#CBD5E1`
- バッジ背景色: `#F1F5F9`
- バッジボーダー色: `#CBD5E1`
- 日付テキスト色: `#64748B`
- 日付テキスト: fontSize 12 / fontWeight SemiBold (w600) / textAlign center
- バッジ角丸: borderRadius 16px（pill型）
- バッジ高さ: 32px
- コンポーネント全高: 48px（バッジ32px + 上下余白各8px）
- 道路帯Canvas側の変更なし（区切りはWidgetとして挿入、Canvasレイヤーはそのまま貫通）
- 全トピック共通で使用できる無彩色系。Teal（Mark色）・Emerald（Link色）と干渉しない

### REQ-DS-04: タイムライン縦線（道路帯）との関係

- 道路帯（グレー帯 + 白破線）はCanvasで全体を通して描画し、区切りウィジェットの挿入によって切断しない
- 区切りウィジェットの高さ（48px）は `_buildTimelineData` 等のY座標計算に加算する
- 区切りはSliverListのアイテムとして既存のインジケーターと同様の仕組みで挿入する

### REQ-DS-05: Mark/Linkカードへの影響

- Markカードの縦幅（72dp）は変更しない
- Linkカードの縦幅（34dp）は変更しない
- 既存のMark/Linkタップ操作（詳細画面への遷移）は影響を受けない

---

## 非機能要件

- `MichiInfoBloc` / Event / State / Repository の構造は変更しない（Widget層のみ変更を原則とする）
- `MarkLinkItemProjection` に日付フィールドが存在しない場合はProjectionへの追加が必要（architectがSpec作成時に確認・判断する）
- `dart analyze` エラー・警告 0 を維持する
- 区切り行のレンダリングがタイムライン全体のスクロールパフォーマンスに影響しないこと（SliverListの既存構造に統合できること）

---

## スコープ外

- `yyyy/MM/dd (曜日)` 表記（曜日付きフォーマット）は Phase 2 以降
- `Day N` 表記（Day 1 / Day 2）は Phase 2 以降
- ダークテーマ対応
- アニメーション（区切りのフェードイン等）
- 代替案B（Tealアクセント）の採用

---

## 受け入れ条件

- [ ] リスト最初のカードの前に必ず区切りが1行表示される
- [ ] 日付が変わるタイミング（前のカードと日付が異なるカードの直前）に区切りが挿入される
- [ ] 同一日付のカードが連続する箇所に区切りが表示されない
- [ ] 区切りの日付テキストが `yyyy/MM/dd` 形式（例: `2024/08/11`）で表示される
- [ ] 横線色 `#CBD5E1`・バッジ背景 `#F1F5F9`・テキスト色 `#64748B` で表示される
- [ ] バッジ高さ32px・コンポーネント全高48px が守られている
- [ ] 道路帯Canvas（グレー帯・白破線）が区切り位置で切断されない
- [ ] Markカード（72dp）・Linkカード（34dp）の縦幅が変化しない
- [ ] 既存のMark/Linkタップ操作が引き続き動作する
- [ ] InsertMode中に区切りが非表示になる
- [ ] `dart analyze` エラー・警告 0
