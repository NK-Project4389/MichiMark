# 要件書: MichiInfo タイムライン 日付セパレーター

作成日: 2026-04-08
作成者: product-manager / architect
バージョン: 1.0
参照デザイン叩き: `docs/Design/draft/2026-04-08_michi_info_date_placement.html`

---

## 概要

MichiInfo「ミチ」タブのタイムラインに、日付変わり目を示す**日付セパレーター行**を挿入する。
1つのイベントに複数日の Mark が記録されるケースで、日付の切れ目が視覚的に判別できない課題を解消する。

採用デザイン案: **候補 B — 日付セパレーター方式**（デザイン提案 `2026-04-08_michi_info_date_placement.html` より）

---

## ユーザーストーリー

- ユーザーとして、ミチタブをスクロールしながら「ここから翌日」と一目で分かるようにしたい
  → 日付が変わるタイミングに水平区切り行が入ることで、旅程の節目をスキャンできる
- ユーザーとして、各 Mark カードに日付を繰り返し表示されても冗長に感じたくない
  → セパレーター1行にまとめることで Mark カード自体の情報密度を維持できる
- ユーザーとして、日帰りイベントでも「いつのドライブか」を確認したい
  → セパレーターは日付が1日のみの場合も必ず先頭に1行表示される

---

## 機能要件

### REQ-DS-01: セパレーター挿入ルール

- タイムラインを上から走査し、**日付が前の Mark と異なる値に変わる直前**にセパレーターを挿入する
- **タイムライン最初の Mark の前にも必ずセパレーターを挿入する**（日付が1種類のみの場合も同様）
- 挿入位置: 直前の Mark/Link カードと次の Mark カードの間

### REQ-DS-02: セパレーターの表示形式

- **基本フォーマット: `4月6日（土）`**（月・日・曜日）
  - 曜日は日本語の括弧付き表記（月〜日）
- **複数年に渡るイベントの場合のみ**: `2025年4月6日（土）` に切り替える（年が現在年と異なる、または Mark 間で年をまたぐ場合）
- `Day N` 表記（Day 1 / Day 2 ...）は **Phase 2 拡張として今回のスコープ外**

### REQ-DS-03: セパレーターのビジュアル

- 背景色: `#EAF5F4`（Teal Tint）
- テキスト色: `#2D6A6A`（Teal Dark）
- 区切り線色: `#B2DFDB`（Timeline Line）
- テキストフォント: 11px / W700
- 背景角丸: 8dp
- セパレーター行の左右に水平区切り線を配置し、中央にラベルを配置するレイアウト（チップ型）
- 既存カラーシステム（Teal 系パレット）のみ使用し、新規カラートークンの追加は不要

### REQ-DS-04: タイムライン縦線との接続

- セパレーター行内では縦線を表示しない（または薄くフェードアウト）
- 次の Mark ドットから新しい縦線を開始することで「節目」を視覚的に表現する

### REQ-DS-05: Mark/Link カードへの影響

- Mark カードの縦幅（`_cardHeight`: 72dp）は変更しない
- Link カードの縦幅（34dp）は変更しない
- 既存の Mark / Link タップ操作（詳細画面への遷移）は影響を受けない

---

## 非機能要件

- `MichiInfoBloc` / Event / State / Projection / Repository の構造は変更しない（Widget 層のみ変更）
- `MarkLinkItemProjection` の日付フィールドを参照してセパレーター挿入判定を行う
  - 日付フィールドが既存 Projection に存在しない場合は Projection への追加が必要（architect が Spec に反映する）
- `dart analyze` エラー・警告 0 を維持する
- セパレーター行のレンダリングがタイムライン全体のスクロールパフォーマンスに影響しないこと（`SliverList` の既存構造に統合できること）

---

## スコープ外

- `Day N` 表記（Day 1 / Day 2）は Phase 2 以降
- ナビバーへの日付範囲表示（B + D ハイブリッド案）は Phase 2 以降
- ダークテーマ対応
- アニメーション（セパレーターのフェードイン等）
- 距離単位の切り替え

---

## 受け入れ条件

- [ ] タイムライン最初の Mark の前に必ずセパレーターが1行表示される
- [ ] 日付が変わるタイミング（前の Mark と日付が異なる Mark の直前）にセパレーターが挿入される
- [ ] セパレーターのテキストが `4月6日（土）` 形式で表示される
- [ ] 複数年にまたがる場合に `2025年4月6日（土）` 形式に切り替わる
- [ ] セパレーター行の背景色が `#EAF5F4`、テキスト色が `#2D6A6A`、区切り線色が `#B2DFDB` で表示される
- [ ] セパレーター行内でタイムライン縦線が非表示（または薄くフェードアウト）になる
- [ ] Mark カード（72dp）・Link カード（34dp）の縦幅が変化しない
- [ ] 既存の Mark / Link タップ操作が引き続き動作する
- [ ] 日帰りイベント（日付が1種類のみ）でセパレーターが先頭に1行のみ表示される
- [ ] `dart analyze` エラー・警告 0
