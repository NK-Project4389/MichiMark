# REQ-splash_michimark_text

## 概要

スプラッシュ画面のロゴ画像の下に「MICHIMARK」テキストを追加表示する。1文字ごとにスペースを空け、フォントは「ヒラギノ丸ゴ」系を使用する。

- **要件ID**: REQ-splash_michimark_text
- **作成日**: 2026-04-20
- **担当**: product-manager
- **ステータス**: Draft（ユーザー確認待ち）
- **種別**: UI改善（UI-25）
- **関連要件書**: REQ-splash_screen_improvement（UI-16）
- **関連Spec**: FS-splash_screen_improvement

---

## ユーザーストーリー

**誰が**: MichiMarkのユーザーが

**何を**: アプリ起動時のスプラッシュ画面で、ロゴの下にブランド名「MICHIMARK」が表示されることで

**なぜ**: アプリ名を視覚的に印象づけ、ブランド認知を強化したい

---

## 要件項目

### テキスト表示

- [ ] スプラッシュ画面のロゴ画像（`splash_logo.png`）の下に「MICHIMARK」テキストを表示する
- [ ] 文字間にスペースを設ける（`letterSpacing` で実装。表示イメージ: "M I C H I M A R K" 相当の間隔）
- [ ] テキスト色は白（`#FFFFFF`）とする（背景色 `#A8D4E6` 上での視認性確保）
- [ ] テキストサイズはロゴ画像（160x160）とのバランスを考慮して設定する（目安: 20〜24sp程度）
- [ ] ロゴ画像とテキストの間に適切な余白を設ける（目安: 16〜24px）

### フォント指定

- [ ] iOS: 「ヒラギノ丸ゴ ProN（HiraMaruProN-W4）」を使用する
- [ ] Android: ヒラギノ丸ゴはAndroidにプリインストールされていないため、フォールバック戦略を定める
  - 案1: カスタムフォントとしてアセットにバンドルする（ライセンス要確認）
  - 案2: Android向けには「Rounded Mplus 1c」等のフリーの丸ゴシックフォントを代替使用する
  - 案3: Googleフォント（`google_fonts`パッケージ）から類似フォントを動的取得する
- [ ] フォントが利用できない場合のフォールバック: システムデフォルトのサンセリフフォント

### アニメーション

- [ ] 既存のフェードイン + スケールアップアニメーションと統一する（ロゴと同じアニメーションを適用）
- [ ] テキストはロゴと同時にアニメーションするか、ロゴよりわずかに遅れてフェードインするか、Architectが設計時に決定する

### テスト対応

- [ ] テキストウィジェットに `Key('splash_text_michimark')` を付与する
- [ ] 既存のスプラッシュテスト（TC-UI16-I001）のテストシナリオにテキスト表示確認を追加する

---

## 受け入れ条件

1. スプラッシュ画面でロゴ画像の下に「MICHIMARK」テキストが表示される
2. 文字間に視認できるスペースが確保されている（letterSpacingによる実装）
3. iOSではヒラギノ丸ゴ系フォントで表示される
4. Androidでもフォールバックにより丸ゴシック系フォントまたはサンセリフフォントで表示される
5. 既存のフェードイン + スケールアニメーションにテキストも含まれる
6. Integration Testの既存スプラッシュスキップパターン（`router.go('/')`）に影響しない

---

## 確認事項（ユーザー確認待ち）

1. **Android対応**: ヒラギノ丸ゴはAndroidにプリインストールされていない。以下のいずれかを選択する必要がある
   - (A) カスタムフォントをアセットにバンドル（アプリサイズ増加。ライセンス確認必要）
   - (B) Android向けに代替フリーフォント（Rounded Mplus 1c等）を使用
   - (C) `google_fonts` パッケージで動的取得（初回表示時に一瞬フォント切り替えが見える可能性）
   - (D) Androidはシステムデフォルトのサンセリフで妥協する
2. **テキスト色**: 背景色 `#A8D4E6` に対して白色で問題ないか（コントラスト比の確認）
3. **letterSpacingの値**: 具体的な数値指定があれば（例: 8.0, 12.0 等）

## 備考

- 現在のスプラッシュ画面の背景色は `#A8D4E6`（アイコンカラー統一対応済み）
- 既存のスプラッシュSpec（FS-splash_screen_improvement）では「表示コンテンツ: ロゴのみ（アプリ名テキストなし）」と記載されているため、本要件でSpecの更新が必要になる
- `letterSpacing` プロパティで文字間スペースを制御する方式が、文字列に手動スペースを挿入する方式より保守性が高い（Architectが設計時に決定）
- フォントのライセンスについては、ヒラギノ丸ゴはApple製品にプリインストールされたフォントであり、iOSでの使用は問題ないが、Androidアプリへのバンドルにはモリサワのライセンスが必要
