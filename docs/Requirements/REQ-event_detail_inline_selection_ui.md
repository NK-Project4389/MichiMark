# 要件書: イベント詳細画面 インライン選択UI

**要件ID**: REQ-event_detail_inline_selection_ui
**作成日**: 2026-04-12
**ステータス**: 承認待ち
**旧要件**: REQ-member_selection_tag_style（廃止・本REQに統合）

---

## 背景・目的

現状、BasicInfo（基本タブ）の交通手段・メンバー・ガソリン支払者、および各Detail画面（MarkDetail・LinkDetail・PaymentDetail）のメンバー選択が `SelectionPage`（フルスクリーン別画面遷移）を使用しており、操作ステップが多い。

また、BasicInfoのタグ入力エリアは「現在のタグ」「新しいタグを追加」「最近使用したタグ」の3ブロックが縦に並んでおり、画面を大きく占有している。

全選択エリアを **画面遷移なし・同一画面内のインラインチップ式** に統一し、タップ数を最小化する。タグとメンバーのチップデザインはカラー・角丸・サイズを統一する。

---

## ユーザーストーリー

- As a ユーザー, I want to 交通手段・メンバー・タグ・支払者をすべて同一画面内で選択したい, so that 別画面に遷移せずに基本情報の設定を完了できる
- As a ユーザー, I want to マーク・リンク・支払いの画面でも画面遷移なしでメンバーを選択したい, so that どの画面でも同じ操作感でメンバーを設定できる
- As a ユーザー, I want to タグ入力欄がコンパクトにまとまっていてほしい, so that 画面の占有面積を減らして他の項目も見やすくしたい

---

## 共通仕様（全エリア共通）

- 画面遷移なし、同一画面内で完結
- `Wrap` ウィジェットで横並び・自動折り返し
- チップが横幅に収まらない場合は次の行へ折り返し
- エリア全体はチップ数に応じて縦に自動伸縮
- **既存のデータモデル・保存処理は変更しない**
- タグ・メンバー・Trans・支払者のチップデザインはカラー・角丸・サイズを統一する

---

## 機能要件

### FR-010: BasicInfo タグ入力エリア改善

**現在の問題**: 「現在のタグ」「新しいタグを追加」「最近使用したタグ」が縦3ブロック構成で画面を大きく占有

#### FR-011: チップ+インライン入力レイアウト
- タグチップと入力欄を同一エリアに横並びで配置（`Wrap`）
- 入力欄はチップの末尾に追従して配置する

#### FR-012: タグチップ削除
- 各チップに「×」ボタンを表示
- ×タップで該当タグを削除する

#### FR-013: 候補ドロップダウン
- 入力欄タップ→フォーカス時に候補リストをドロップダウン表示
- 未入力状態でも「最近使用したタグ」を候補として表示
- 文字入力時は前方一致でリアルタイムフィルタリング
- 候補は最大3〜4件表示（スクロール可）
- **マスタに存在するタグ** → 候補タップで即チップ追加
- **マスタ未登録の文字列** → 候補欄の最下部に「"xxx" を追加」を表示 → タップで新規チップ生成
- 候補タップ後: チップ追加 → 入力欄クリア → ドロップダウン閉じる

---

### FR-020: BasicInfo 交通手段 チップ式切り替え

**現在の問題**: `_SelectionRow`（別画面遷移）を使用しており操作ステップが多い

#### FR-021: 登録済み交通手段を全件チップ表示
- 交通手段マスタに登録されている全交通手段をチップとして横並びで表示
- **未選択**: アウトライン表示（枠線のみ）
- **選択済み**: 塗りつぶし表示＋チェックマーク（✓）
- タップでON/OFF即時切り替え（single選択：他の選択は自動解除）

#### FR-022: 画面表示時の初期状態
- 既存の選択済み交通手段は選択状態で表示する

---

### FR-030: BasicInfo メンバー選択 インライン化

**現在の問題**: `_SelectionRow`（別画面遷移）を使用しており操作ステップが多い

#### FR-031: チップ+インライン入力レイアウト
- 選択済みメンバーは `Chip`（塗りつぶし＋×ボタン）で横並び表示
- 入力欄はチップの末尾に追従して配置する

#### FR-032: サジェストドロップダウン
- 入力欄フォーカス時に候補リストをドロップダウン表示
- 未入力状態: 「最近選択したメンバー」（過去イベント使用メンバー）を候補として表示
- 文字入力時: 部分一致でメンバーマスタを検索しリアルタイムフィルタリング
- 既に選択済みのメンバーはサジェストから除外する
- **マスタに存在するメンバー** → 候補タップで即チップ追加
- **マスタ未登録の名前** → 候補欄の最下部に「"xxx" を追加」を表示 → タップでマスタ登録+チップ追加

#### FR-033: 重複チェック
- 既に選択済みと同名を追加しようとした場合は無視する

---

### FR-040: BasicInfo ガソリン支払者 チップ式切り替え

**現在の問題**: `_SelectionRow`（別画面遷移）を使用しており操作ステップが多い

#### FR-041: イベントメンバーをチップ表示
- BasicInfoで選択済みのイベントメンバーを全員チップとして横並びで表示
- **未選択**: アウトライン表示（枠線のみ）
- **選択済み**: 塗りつぶし表示＋チェックマーク（✓）
- タップでON/OFF即時切り替え（single選択：他の選択は自動解除）
- イベントメンバーが0人の場合はチップを表示せず「メンバーを先に選択してください」などのヒントを表示

#### FR-042: 画面表示時の初期状態
- 既存の選択済み支払者は選択状態で表示する

---

### FR-050: MarkDetail メンバー選択 インライン化

**現在の問題**: `context.push<SelectionResult>`（別画面遷移）を使用

#### FR-051: イベントメンバーをチップ表示
- MarkDetailのメンバー選択行を、イベントメンバー全員のチップ表示に変更
- **未選択**: アウトライン表示
- **選択済み**: 塗りつぶし＋✓
- タップでON/OFF即時切り替え（multiple）

#### FR-052: 画面表示時の初期状態
- 既存の選択済みメンバーは選択状態で表示する

---

### FR-060: LinkDetail メンバー選択 インライン化

FR-051・FR-052と同様の仕様をLinkDetailに適用する。

---

### FR-070: PaymentDetail 支払者・割り勘メンバー インライン化

**現在の問題**: `context.push<SelectionResult>`（別画面遷移）を使用

#### FR-071: 支払者チップ（single選択）
- イベントメンバー全員をチップとして表示
- タップでON/OFF切り替え（single選択）

#### FR-072: 割り勘メンバーチップ（multiple選択）
- イベントメンバー全員をチップとして表示
- タップでON/OFF切り替え（multiple選択）
- 支払者（payMember）は常にON固定・非活性化（既存仕様維持）

---

## 非機能要件

- メンバーマスタ・タグマスタは画面読み込み時にキャッシュし、入力変化のたびにDBを叩かない
- 検索は大文字・小文字を区別しない
- メンバーサジェストは直近10イベント程度を参照（パフォーマンス考慮）

## 影響範囲

### Phase A: BasicInfo

| ファイル | 変更種別 |
|---|---|
| `basic_info_view.dart` | `_SelectionRow`（Trans・Members・PayMember）を各インラインセクションに置き換え、`_TagInputSection`を改善版に更新 |
| `basic_info_event.dart` | Trans/Member/Tag各インタラクションイベント追加 |
| `basic_info_state.dart` | `transSuggestions`・`memberSuggestions`・`tagSuggestions`フィールド整理 |
| `basic_info_bloc.dart` | 各ハンドラ実装 |

### Phase B: 各Detail画面

| ファイル | 変更種別 |
|---|---|
| `mark_detail_page.dart` | メンバー選択を `context.push` からインラインチップに変更 |
| `link_detail_page.dart` | メンバー選択を `context.push` からインラインチップに変更 |
| `payment_detail_page.dart` | 支払者・割り勘メンバー選択を `context.push` からインラインチップに変更 |
| 各 bloc/event/state | インライン選択用イベント・状態追加 |

## 対応しないもの

- SelectionPage の完全削除（`eventTopic`・`markActions`・`linkActions` は別画面遷移を維持）
- メンバーマスタ設定画面（member_setting）の変更
- タグマスタ設定画面の変更
